| :--- | :--- | :--- | :--- |
| **Lab1: 五级流水线与转发** | **五级流水线基本架构** | 构建 IF、ID、EX、MEM、WB 五个经典流水段，实现指令在段间寄存器的稳定流动 [1, 2]。 | [1], [3], [2] |
| | **RAW 数据冒险与转发机制** | 验证真数据相关（RAW），通过从 EX/MEM 或 MEM/WB 阶段前递数据解决冒险，避免流水线断流 [4, 5]。 | [5], [4], [2] |
| | **流水线停顿控制** | 在数据相关无法通过转发解决或取指未就绪时，通过保持 PC 和 IF/ID 寄存器不变来插入“气泡” [4, 6]。 | [6], [4] |
| **Lab2: Load/Store 访存** | **Load/Store 指令语义** | 实现不同宽度的内存读写指令（LB, LH, LW, LD 等），处理符号扩展与零扩展 [7, 8]。 | [8], [7] |
| | **内存总线握手协议** | 验证 `valid` 请求与 `dataOk` 响应的握手逻辑，要求在响应到来前维持信号稳定 [9, 10]。 | [10], [6], [9] |
| | **地址对齐与字节使能** | 处理硬件自动忽略地址低 3 位（8 字节对齐）的情况，通过 `strobe` 信号精确操作特定字节 [9, 11]。 | [11], [9] |
| **Lab3: 分支与跳转** | **控制流指令语义** | 实现条件分支（B型）与无条件跳转（JAL/JALR），处理 PC 相对寻址与寄存器间接寻址 [12, 13]。 | [12], [13] |
| | **控制冒险与流水线冲刷** | 验证在 EX 阶段确定跳转后，必须清空（Flush）错误路径上已经进入流水线的指令（IF/ID, ID/EX） [14, 15]。 | [16], [15], [14] |
| | **内存映射 I/O (MMIO)** | 验证通过特定内存地址访问外设的机制，并在 Difftest 框架中对这些地址实现跳过（Skip）比对 [17, 18]。 | [18], [17] |

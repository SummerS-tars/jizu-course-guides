以下是基于Week 3课程记录、课件04及相关实验资料，对CISC与RISC设计哲学、RISC-V指令格式及Load/Store架构的对比总结：

### 1. CISC (x86) 与 RISC (RISC-V) 设计哲学对比

| 维度 | CISC (x86) | RISC (RISC-V) |
| :--- | :--- | :--- |
| **核心目标** | 强化指令功能，减少程序指令条数，支持高级语言 [1, 2] | 简化指令系统，追求高效流水线实现，降低CPI [1, 3] |
| **指令长度** | 可变长度（1-15字节），编码复杂 [4, 5] | 固定长度（通常32位），编码规整 [4, 6] |
| **寻址方式** | 种类繁多且复杂（如变址、间址等） [2, 7] | 种类少且简单（如寄存器、立即数、偏移寻址） [3, 8] |
| **访存限制** | 运算指令可直接访问存储器 [2, 7] | 仅Load/Store指令访存，运算指令仅限寄存器 [3, 9] |
| **控制逻辑** | 大多采用微程序控制器 [7, 10] | 采用组合逻辑/硬布线控制 [3, 11] |
| **寄存器** | 专用寄存器较多，通用寄存器较少 [7, 12] | 配置大量通用寄存器 [11, 13] |

### 2. RISC-V 六种基础指令格式 (RV32I)

RISC-V通过固定寄存器索引位置（rs1, rs2, rd）来简化译码电路设计 [14, 15]。

| 类型 | 主要用途 | 字段构成 (31:0) | 备注 |
| :--- | :--- | :--- | :--- |
| **R型** | 寄存器-寄存器运算 | funct7, rs2, rs1, funct3, rd, opcode | 用于ADD, SUB, AND, OR等 [14, 16] |
| **I型** | 立即数运算 / Load / JALR | imm[11:0], rs1, funct3, rd, opcode | 12位立即数需符号扩展 [6, 14] |
| **S型** | Store 指令 | imm[11:5], rs2, rs1, funct3, imm[4:0], opcode | 立即数拆分以保持rs1/rs2位置固定 [15, 17] |
| **B型** | 条件分支 (Branch) | imm[12\|10:5], rs2, rs1, funct3, imm[4:1\|11], opcode | 偏移量需乘以2（16位对齐） [16, 17] |
| **U型** | 长立即数加载 (LUI/AUIPC) | imm[31:12], rd, opcode | 加载20位高位立即数 [14, 16] |
| **J型** | 无条件跳转 (JAL) | imm[20\|10:1\|11\|19:12], rd, opcode | 跳转范围较广，采用打乱的立即数编码 [14, 16] |

### 3. Load/Store 架构

RISC-V 采用典型的 **Load/Store 架构**，这是其简化硬件设计的核心 [9, 18]。

*   **基本原则**：运算类指令（如加减、逻辑运算）的操作数必须来自寄存器，不能直接操作内存数据 [19, 20]。
*   **内存访问**：只有专门的 `load`（从内存读入寄存器）和 `store`（将寄存器存入内存）指令可以访问存储器 [2, 21]。
*   **设计优势**：
    *   **简化相关性判断**：CPU只需通过寄存器号即可判断指令间的数据相关，复杂的地址比较仅在执行访存时进行，有利于乱序执行和多发射技术 [22, 23]。
    *   **规整执行时间**：由于访存与运算分离，指令执行时间更趋一致，便于流水线化 [2, 24]。
*   **实验应用 (Wiki-Lab2/3)**：Lab 2 侧重于实现基本的访存指令（如 `lw`, `sw`），而 Lab 3 则在此基础上完善跳转与分支指令，体现了取指、译码与访存阶段的分离 [25, 26]。

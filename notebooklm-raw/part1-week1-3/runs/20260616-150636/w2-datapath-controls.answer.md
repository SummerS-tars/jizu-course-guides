在单周期 CPU 设计中，所有指令必须在一个时钟周期内完成执行 [1]。以下是基于课件与实验报告的核心解析：

### 1. 数据通路核心模块
*   **程序计数器 (PC)**：一个 32 位时序逻辑寄存器，用于存储当前指令地址，并在每个时钟上升沿更新 [1, 2]。
*   **指令存储器 (IM)**：组合逻辑部件（只读），根据 PC 提供的地址立即输出对应的 32 位指令码 [3, 4]。
*   **寄存器堆 (Register File)**：包含 32 个通用寄存器。拥有两个异步读端口（根据地址立即输出数据）和一个同步写端口（仅在时钟上升沿且写使能有效时写入）[2, 3]。
*   **算术逻辑单元 (ALU)**：纯组合逻辑部件，根据控制信号对输入的两个操作数进行算术或逻辑运算，并输出结果及零标志位 (Zero) [2, 3]。
*   **数据存储器 (DM)**：用于存储数据。读操作为组合逻辑，写操作为时序逻辑（需时钟触发）[3]。
*   **控制单元 (CU)**：负责对指令的操作码 (Opcode) 和功能码 (Funct) 进行译码，生成指挥数据流向的所有控制信号 [3, 5]。

### 2. 核心控制信号定义
*   **RegWrite**：写使能信号。为 1 时，允许将数据写入目标寄存器（如 R 型指令、`lw`）；为 0 时禁止写入（如 `sw`、`beq`）[6-8]。
*   **ALUSrc**：选择 ALU 的第二个操作数来源。0 表示来自寄存器堆的第二个输出 (Bus B)；1 表示来自指令中的立即数扩展值 [6-8]。
*   **MemtoReg**：选择写回寄存器的数据源。0 表示选择 ALU 的计算结果；1 表示选择从数据存储器读出的数据 [6-8]。
*   **Branch**：分支控制信号。为 1 且 ALU 输出的 `Zero` 为真时，表示 `beq` 条件成立，PC 将跳转至目标地址 [7-9]。

### 3. Load 指令为何决定关键路径
Load 指令（`lw`）是单周期 CPU 中延迟最长的指令，因为它必须经历五个完整的步骤 [10-12]：
1.  **取指**：访问指令存储器。
2.  **译码/读寄存器**：访问寄存器堆。
3.  **执行**：通过 ALU 计算内存地址。
4.  **访存**：访问数据存储器读取数据。
5.  **写回**：将读出的数据写回寄存器堆。

由于单周期 CPU 的**时钟周期必须大于等于最长指令的延迟** [13]，因此 `lw` 指令的物理延迟直接限制了处理器的最高主频，导致执行较快指令（如 R 型或跳转指令）时存在大量资源闲置 [14, 15]。

根据课件 7a 及复习笔记，互连网络的基础分类、评价准则及核心计算总结如下：

### 1. 互连网络基础分类
互连网络通过以下三个维度进行描述：
*   **定时方式 (Timing)**：
    *   **同步系统**：使用统一时钟信号，所有结点同步工作（如 SIMD 阵列处理机）[1, 2]。
    *   **异步系统**：没有统一时钟，各结点独立工作，通过握手协议或消息传递通信 [1, 2]。
*   **交换方法 (Switching)**：
    *   **电路交换**：在源与目的间建立专用物理通路，通信期间一直保持，延迟低但线路利用率可能较低 [1-3]。
    *   **分组交换**：将信息分割成“包”（Packet），在网络中存储转发，资源复用率高，是现代数据中心的主流 [1, 2, 4]。
*   **控制策略 (Control)**：
    *   **集中控制**：由全局控制器接收请求并设置开关，适用于小规模、拓扑固定的集群 [1, 2]。
    *   **分散控制**：不存在全局控制器，由互连网络各结点分散处理请求，扩展性强（如以太网） [1, 5]。

### 2. 核心评价准则定义
*   **网络规模 ($N$)**：网络中结点的总数 [5, 6]。
*   **结点度 ($d$)**：与结点相连的边数。入边为入度，出边为出度。度越大，成本越高 [5, 7, 8]。
*   **网络直径 ($D$)**：任意两结点间距离（跨越边数）的最大值。代表最坏情况下的通信延迟，越小越好 [5, 7, 8]。
*   **对剖带宽 (Bisection Bandwidth, $B$)**：将网络切成相等的两半时，切口处最小边数乘以每条边的带宽。反映网络最大流量瓶颈 [5, 7, 8]。
*   **平均延迟 (Average Latency)**：在均匀流量模式下，所有结点对之间最短路径长度的平均值 [8]。

### 3. 数值计算例：$n \times n$ Mesh vs. Torus (环网)
假设网络规模 $N = n \times n = 64$（即 $n=8$），每条链路带宽为 $1$。

#### **A. 二维网格 (2D Mesh)**
1.  **结点度 ($d$)**：不对称。内部结点为 4，边缘为 3，四个角结点为 2 [9, 10]。
2.  **网络直径 ($D$)**：从左上角 $(0,0)$ 到右下角 $(n-1, n-1)$。
    *   计算：$D = (n-1) + (n-1) = 2(n-1) = 2 \times (8-1) = \mathbf{14}$ [9, 10]。
3.  **对剖带宽 ($B$)**：垂直切开中间的一排垂直边。
    *   计算：$B = n = \mathbf{8}$ [11, 12]。

#### **B. 二维环网 (2D Torus)**
1.  **结点度 ($d$)**：对称。由于增加了回绕链路（Wrap-around），所有结点度均为 $\mathbf{4}$ [9, 10]。
2.  **网络直径 ($D$)**：由于有回绕边，每个维度最远跳数为 $\lfloor n/2 \rfloor$。
    *   计算：$D = \lfloor n/2 \rfloor + \lfloor n/2 \rfloor = 2 \times (8/2) = \mathbf{8}$ [9, 10]。
3.  **对剖带宽 ($B$)**：垂直切开时，除了中间 $n$ 条边，还有 $n$ 条回绕边被切断。
    *   计算：$B = 2n = 2 \times 8 = \mathbf{16}$ [11]。
4.  **均匀流量下平均延迟**（按 Week 16 复习逻辑）：
    *   单维双向环（$n$ 个结点）的平均距离约为 $n/4$。
    *   二维环网即两个维度的平均距离相加：$Latency \approx n/4 + n/4 = 8/4 + 8/4 = \mathbf{4}$ 跳 [8]。

**总结提示**：期末计算重点在于**曼哈顿距离**的推导，Mesh 是 $2(n-1)$，Torus 因为有回绕边，直径和延迟通常约减半 [8]。

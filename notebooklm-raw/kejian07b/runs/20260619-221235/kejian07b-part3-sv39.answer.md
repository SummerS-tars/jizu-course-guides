### 1. 虚拟地址空间与 Sv39 地址格式
在 RISC-V Sv39 模式下，虚拟地址（VA）为 **39 位**，物理地址（PA）为 **56 位** [1, 2]。

*   **虚拟地址 (VA) 划分** [1, 3]：
    *   **VPN[4] (9位)**：第一级页表（根页表）索引。
    *   **VPN[5] (9位)**：第二级页表索引。
    *   **VPN (9位)**：第三级页表索引。
    *   **Page Offset (12位)**：页内偏移量（对应 4KB 的页大小）。
*   **物理地址 (PA) 划分** [1, 6]：
    *   **PPN (44位)**：物理页号。
    *   **Page Offset (12位)**：与虚拟地址偏移量一致。

### 2. PTE（页表项）各位含义
Sv39 的页表项为 **64 位**，包含物理页号和控制位 [1, 7]：

*   **V (Valid)**：有效位。若 V=0，访问将触发 **缺页异常 (Page Fault)** [7]。
*   **R/W/X**：读/写/执行权限。全 0 表示该项指向下一级页表；非全 0 表示叶子节点 [8]。
*   **U (User)**：用户态位。U=1 表示用户模式可访问 [8]。
*   **A (Accessed)**：访问位。指示页面自上次清除以来是否被访问过 [8]。
*   **D (Dirty)**：脏位。指示页面是否被写过（用于回写策略） [8]。
*   **PPN (44位)**：存储最终的物理页号或下一级页表的基地址 [8]。

### 3. satp 寄存器与 WARL 机制
`satp` (Supervisor Address Translation and Protection) 是控制地址转换的核心 CSR [9, 10]。

*   **字段定义** [9]：
    *   **MODE (4位)**：`0` 为直连模式（Bare），`8` 为启用 Sv39。
    *   **ASID (16位)**：地址空间标识符，用于区分进程，减少 TLB 冲刷 [10]。
    *   **PPN (44位)**：**根页表**的物理页号。
*   **WARL 机制**：即 "Write Any Values, Reads Legal Values"。允许写入任意值，但读回时仅返回硬件支持的合法值。例如若不支持某些位，硬件会自动忽略写入 [10]。

### 4. TLB 原理与页表遍历
**TLB (快表)** 是页表的专用 Cache，存储 VPN 到 PPN 的映射 [11-13]。

*   **TLB Hit**：直接从硬件获取物理页号，无需访存查表 [12]。
*   **TLB Miss**：硬件或软件启动 **PTW (页表遍历器)** [14, 15]。
*   **三级遍历逻辑** [3, 16]：
    1.  从 `satp.PPN` 得到根页表基址。
    2.  地址 = 基址 + `VPN[i] * 8`（每个 PTE 占 8 字节）。
    3.  读取 PTE，获取下一级物理页号，重复直至找到叶子节点。
    4.  **最终物理地址 = 叶子 PTE.PPN 拼接 VA.Offset** [16, 17]。

### 5. 完整地址转换数值例
假设给定：
*   **satp** = `0x8000000000001000` (MODE=8, PPN=`0x1000`)
*   **虚拟地址 (VA)** = `0x0000002040608ABC`

**第一步：拆分 VA**
*   二进制扩展后提取：
    *   `VPN[4]` = `0x81` (高 9 位：`38:30`)
    *   `VPN[5]` = `0x2` (中间 9 位：`29:21`)
    *   `VPN` = `0x8` (低 9 位：`20:12`)
    *   `Offset` = `0xABC` (页内偏移：`11:0`)

**第二步：计算各级 PTE 地址**
1.  **一级（根）页表**：
    *   基址 = `satp.PPN << 12` = `0x1000000`。
    *   PTE 地址 = `0x1000000 + (VPN[4] * 8)` = `0x1000000 + 0x408` = `0x1000408`。
    *   假设读取该地址 PTE 内容，得到 `PPN = 0x2000`。
2.  **二级页表**：
    *   基址 = `0x2000 << 12` = `0x2000000`。
    *   PTE 地址 = `0x2000000 + (VPN[5] * 8)` = `0x2000000 + 0x10` = `0x2000010`。
    *   假设读取得到 `PPN = 0x3000`。
3.  **三级页表**：
    *   基址 = `0x3000 << 12` = `0x3000000`。
    *   PTE 地址 = `0x3000000 + (VPN * 8)` = `0x3000000 + 0x40` = `0x3000040`。
    *   假设读取得到叶子 PTE，其 `PPN = 0x5555`。

**第三步：计算最终物理地址 (PA)**
*   **PA** = `(0x5555 << 12) | 0xABC` = `0x5555ABC`。

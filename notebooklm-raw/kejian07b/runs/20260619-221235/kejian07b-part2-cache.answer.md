### 1. 局部性原理与 Cache 工作原理

Cache 设计的核心逻辑在于利用**程序的局部性原理 (Locality)** [1, 2]：
*   **时间局部性 (Temporal Locality)**：刚访问过的数据很可能不久后再次访问（如循环变量 `sum`）[1, 3]。
*   **空间局部性 (Spatial Locality)**：刚访问过单元的邻近单元很可能不久后被访问（如数组遍历 `A[i], A[i+1]`）[1, 3]。

**工作原理**：Cache 位于 CPU 和主存之间，采用快速的 SRAM 介质 [2, 4]。当 CPU 发出访存请求时，硬件先检查地址所在的数据块是否已在 Cache 中（命中 Hit），若在则直接读取；若不在（缺失 Miss），则从主存调入整个数据块到 Cache [4, 5]。

---

### 2. 地址拆分规则 (Tag/Index/Offset)

为了在 Cache 中定位数据，物理地址被逻辑上拆分为三部分 [6, 7]：

*   **Offset (块内偏移)**：确定数据在 Cache 块（Block/Line）内的具体字节位置 [7, 8]。
    *   $位数 = \log_2(\text{块大小 Block Size})$ [6, 9]。
*   **Index (索引)**：确定数据映射到 Cache 的哪一行或哪一组 [6, 9]。
    *   直接映射：$位数 = \log_2(\text{行数})$ [6, 9]。
    *   组相联：$位数 = \log_2(\text{组数})$ [10, 11]。
    *   全相联：无 Index 位 [10, 12]。
*   **Tag (标记)**：用于区分映射到相同索引的不同主存块，存储在 Cache 的元数据中 [7, 13]。
    *   $位数 = \text{物理地址总位数} - \text{Index 位数} - \text{Offset 位数}$ [6]。

---

### 3. AMAT 平均访存时间手算

**AMAT (Average Memory Access Time)** 公式衡量存储系统的性能 [14, 15]：
$$\text{AMAT} = \text{命中时间 (Hit Time)} + \text{缺失率 (Miss Rate)} \times \text{缺失代价 (Miss Penalty)}$$
*（来源：课件7b [14] / 唐朔飞第4章 [16] / P&H第5章 [17]）*

**手算示例**：
若 Cache 访问需 1 个周期，缺失率为 5%，主存访问（缺失代价）需 100 个周期，则：
$\text{AMAT} = 1 + 0.05 \times 100 = 6 \text{ 个周期}$ [18]。

### 4. 完整数值例题

**场景设定**：
*   **Cache 参数**：容量 32KB，4 路组相联 (4-way set associative)，块大小 64B，物理地址 32 位 [9, 19]。
*   **待查物理地址**：`0x00012345`。

**第一步：计算地址字段位数**
1.  **Offset**：$\log_2(64) = 6$ 位（即物理地址的 [5:0] 位）[6, 9]。
2.  **组数 (Sets)**：
    *   总块数 = $32\text{KB} / 64\text{B} = 512$ 块 [20]。
    *   组数 = $512 / 4\text{路} = 128$ 组 [10, 20]。
3.  **Index**：$\log_2(128) = 7$ 位（即物理地址的 [12:6] 位）[10, 11]。
4.  **Tag**：$32 - 7 - 6 = 19$ 位（即物理地址的 [31:13] 位）[6]。

**第二步：拆分地址 `0x00012345`**
二进制展示：`0000 0000 0000 0001 0010 0011 0100 0101`
1.  **Offset** (低 6 位)：`000101` $\rightarrow$ **0x05**。
2.  **Index** (接下来的 7 位)：`0001000` $\rightarrow$ **0x08**（十进制第 8 组）。
3.  **Tag** (高 19 位)：`0x00012 >> 1 = 0x00009`（即剩余高位部分）。

**第三步：命中判断与状态机 (Lab4 背景)** [20, 21]
1.  **IDLE**: 接收 `0x00012345`。
2.  **COMPARE_TAG**: 硬件根据 **Index 0x08** 索引到 Cache 的第 8 组，并行提取该组内 4 路的 Tag 值 [22]。
3.  **判定**: 将地址中的 **Tag 0x00009** 与这 4 个 Tag 比较，并检查**有效位 (Valid)** [22, 23]。
    *   若匹配且有效，则 **Hit**，根据 **Offset 0x05** 返回数据。
    *   若均不匹配，则 **Miss**，进入 **ALLOCATE/REPLACE** 状态，若选中的行被修改过（Dirty），则需先 **WRITE_BACK** 写回内存，再从内存加载新块 [20, 21]。

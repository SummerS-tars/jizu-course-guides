根据**课件05 中央处理器**及相关课程笔记，单周期处理器（Single Cycle Processor）的设计核心在于每条指令都在一个时钟周期内完成执行 [1-3]。

### 1. CPU 功能与指令执行步骤
CPU 的基本职能是周而复始地执行指令 [4]。单周期下的执行步骤通常分为以下五个阶段 [1, 5, 6]：
*   **取指令 (IF)**：根据程序计数器（PC）中的地址，从指令存储器中读取指令，同时计算 `PC+4` [1, 5]。
*   **指令译码 (ID)**：解析指令操作码（Opcode），生成控制信号，并从寄存器堆中读取源操作数 [1, 5]。
*   **执行运算 (EX)**：利用算术逻辑单元（ALU）进行算术、逻辑运算或计算有效地址 [1, 5]。
*   **访存 (MEM)**：若是访存指令（Load/Store），则读写数据存储器；其他指令此阶段为空 [1, 5]。
*   **写回结果 (WB)**：将运算结果或访存读取的数据写回目的寄存器 [1, 5]。

### 2. 数据通路的基本组成
数据通路（Datapath）由执行指令所需的元件通过总线或分散方式连接而成 [7-9]：
*   **组合逻辑元件（操作元件）**：
    *   **ALU**：核心运算部件，执行加减、与或等操作 [8, 9]。
    *   **多路选择器 (MUX)**：根据控制信号（如 `ALUSrc`）在多个输入中选择路径 [8, 10]。
    *   **扩展器 (Extender)**：将16位立即数进行符号扩展或零扩展至32位 [8, 11]。
*   **时序逻辑元件（状态元件）**：
    *   **PC**：保存当前指令地址，在时钟边沿更新 [12, 13]。
    *   **寄存器堆 (Register File)**：包含通用寄存器，支持双端口异步读和单端口同步写 [5, 14]。
    *   **存储器接口**：单周期通常将指令存储器（ROM）与数据存储器（RAM）物理分离，以避免结构冒险 [14, 15]。

### 3. 三类指令的数据流（以 MIPS 为例）

| 指令类型 | 数据通路流向描述 | RISC-V Lab1 对照点 |
| :--- | :--- | :--- |
| **R型 (如 add)** | `Reg[rs], Reg[rt] -> ALU -> 写回 Reg[rd]`。由 `ALUOp` 控制计算，`RegWr=1` [5, 16]。 | RISC-V 的 `rd` 字段固定在位 [11:7]，与 MIPS 位置不同 [14, 17]。 |
| **Load/Store** | `Reg[rs] + 符号扩展(imm) -> ALU (算地址) -> 读/写数据存储器`。`lw` 写回 `Reg[rt]` [10, 18, 19]。 | RISC-V 的立即数采用零散编码（Immediate Splitting）以简化硬件连接 [17, 20]。 |
| **分支 (beq)** | `Reg[rs], Reg[rt] -> ALU (做减法) -> Zero标志位 -> 下地址逻辑控制 PC` [10, 21]。 | RISC-V 分支目标地址计算是基于当前指令 `PC`，而非 `PC+4` [17, 22]。 |

**提示**：在单周期设计中，由于所有操作必须在一周期内跑完，时钟周期由最慢的 `lw` 指令决定，这导致了较低的资源利用率 [3, 23]。

以下是基于**课件 05 中央处理器**及相关参考资料总结的 5 组易混淆概念对比：

### 1. 处理器实现方式对比
| 特性 | 单周期 (Single-cycle) | 多周期 (Multi-cycle) | 流水线 (Pipelining) |
| :--- | :--- | :--- | :--- |
| **指令周期** | 一个时钟周期完成一条指令 [1, 2] | 多个时钟周期完成一条指令 [1, 2] | 多个指令重叠执行，理想 CPI 为 1 [3, 4] |
| **时钟频率** | 较低（由最慢指令决定）[1, 5] | 较高（由最慢阶段决定）[1, 5] | 较高（取决于流水段平衡）[3, 4] |
| **资源复用** | 不可复用（需独立 IM/DM）[1, 2] | 可复用（如合用 IM/DM、ALU）[1, 2] | 不可复用（存在结构冒险）[6, 7] |
| **中间寄存器** | 无需临时寄存器 [1, 2] | 需临时寄存器（如 IR, MDR, A, B）[1, 2] | 需流水段寄存器（如 IF/ID）[8, 9] |
*(来源：课件05 / Week6 / 笔记-08)*

### 2. 逻辑元件角色对比
| 特性 | 组合逻辑 (Combinational) | 时序逻辑 (Sequential) |
| :--- | :--- | :--- |
| **功能** | 进行数据处理和运算（操作元件）[10, 11] | 进行信息存储和状态保存（存储元件）[11, 12] |
| **输出特性** | 仅取决于当前输入，无记忆性 [10, 13] | 取决于输入及先前状态，具有记忆性 [14, 15] |
| **典型部件** | ALU、加法器、多路选择器、译码器 [10, 16] | PC、寄存器堆、存储器、触发器 [11, 16] |
| **定时控制** | 不需要时钟信号定时 [10, 17] | 由时钟信号控制写入时间 [14, 17] |
*(来源：课件05 / Week2 / 笔记-Week01)*

### 3. 控制器实现方式对比
| 特性 | 硬连线控制 (Hardwired) | 微程序控制 (Microprogrammed) |
| :--- | :--- | :--- |
| **逻辑基础** | 组合逻辑电路（PLA/FSM）[18, 19] | 存储逻辑（只读存储器 ROM/控存）[18, 20] |
| **优点** | 速度快，适合规整的 RISC 指令集 [21, 22] | 设计灵活、规整、易于修改和扩展 [20, 23] |
| **缺点** | 结构庞杂，难以维护或增删指令 [20, 22] | 速度慢（需访问控存取微指令）[20, 23] |
| **控制信号** | 每个时钟周期产生一组新值 [19, 24] | 对应一条或多条微指令 [19, 24] |
*(来源：课件05 / Week7 / 笔记-08)*

### 4. 正常流中断事件对比
| 特性 | 异常 (Exception / 同步) | 中断 (Interrupt / 异步) |
| :--- | :--- | :--- |
| **来源** | CPU 内部，与当前执行的指令相关 [25, 26] | CPU 外部，由 I/O 设备等触发 [25, 26] |
| **同步性** | 同步，发生在指令执行期间 [26, 27] | 异步，可能在任何时刻到达 [26, 27] |
| **典型例子** | 溢出、除数为0、非法指令、缺页 [25, 28] | 定时器中断、外设请求（如键盘输入）[25, 28] |
| **处理时机** | 发现异常立即停止当前指令 [29, 30] | 通常在每条指令执行结束时检测 [31, 32] |
*(来源：课件05 / Week9 / Lab6)*

### 5. CPU 基本组成对比
| 特性 | 数据通路 (Datapath) | 控制器 (Control Unit) |
| :--- | :--- | :--- |
| **角色定义** | 指令的“执行部件”，如管弦乐团 [12, 33] | 指令的“指挥部件”，如指挥家 [12, 33] |
| **主要任务** | 进行数据的存储、处理和传送 [10, 34] | 对指令译码，生成各部件的控制信号 [12, 35] |
| **流动信息** | 操作数、运算结果、PC 值、指令码 [36, 37] | 控制信号（如 RegWrite, ALUSrc）[37, 38] |
| **设计核心** | 算逻部件、寄存器堆及互连路径 [13, 39] | 状态转换图、逻辑表达式或微程序 [40, 41] |
*(来源：课件05 / Week2 / Lab1)*

### SATP 寄存器与虚拟内存控制

在 RISC-V 的 Sv39 模式下，**SATP (Supervisor Address Translation and Protection)** 寄存器是控制虚拟内存管理的核心 [1]。

#### 1. SATP 寄存器字段 (Sv39 模式)
| 字段 | 位数 | 含义 |
| :--- | :--- | :--- |
| **MODE** | 63-60 | 控制地址翻译模式：0 为关闭 (Bare)，8 为开启 Sv39 模式 [1, 2]。 |
| **ASID** | 59-44 | **地址空间标识符**，用于区分不同进程，减少上下文切换时的 TLB 冲刷 [1, 2]。 |
| **PPN** | 43-0 | **根页表物理页号**，指向一级页表的物理起始地址 [1-3]。 |

#### 2. WARL 机制
*   **含义**：全称“Write Any Values, Reads Legal Values” [1]。
*   **行为**：允许软件写入任何值，但硬件读回时仅返回合法值 [1]。
*   **特性**：硬件会自动忽略不支持的位（例如在 Sv39 下 Mode 字段仅支持 0 和 8） [1]。

#### 3. TLB 的作用
*   **加速翻译**：TLB 是页表项 (PTE) 的高速缓存，存储最近使用的虚实地址映射 [4, 5]。
*   **性能优化**：CPU 访存时首先查询 TLB，命中则无需访问主存中的多级页表，显著提升速度 [5-7]。

#### 4. SFENCE.VMA 的失效范围
该指令用于同步 TLB 缓存，通过 `rs1` (虚拟地址) 和 `rs2` (ASID) 的组合确定清理范围 [8]：
*   **rs1=x0, rs2=x0**：失效**所有**地址空间中的所有虚拟地址条目 [8]。
*   **rs1=x0, rs2≠x0**：失效由 `rs2` 指定的 **ASID 对应空间**内的所有条目 [8]。
*   **rs1≠x0, rs2=x0**：失效**当前虚拟地址**在所有地址空间中的条目（常用于全局页面） [8]。
*   **rs1≠x0, rs2≠x0**：失效**指定 ASID 空间**中**指定虚拟地址**的条目 [8]。

#### 5. ASID 回收场景
*   **触发条件**：当可分配的 ASID 资源耗尽时，操作系统需要回收并重新分配 [1]。
*   **同步要求**：在 ASID 资源回收并重新映射时，操作系统必须执行 `SFENCE.VMA` 指令来同步 TLB，防止读取到过时的翻译条目 [1, 9]。

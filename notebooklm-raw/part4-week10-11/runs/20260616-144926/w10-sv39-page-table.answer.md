RISC-V Sv39 三级页表是用于 64 位系统的虚拟内存管理方案，其核心是将虚拟地址通过三级页表映射为物理地址。

### 1. Sv39 地址结构与位域划分
Sv39 方案支持 39 位的虚拟地址（VA）和 56 位的物理地址（PA）[1, 2]。

*   **虚拟地址 (VA, 39位)**：划分为三级虚拟页号（VPN）和页内偏移量 [1, 2]。
    *   `[38:30]`：**VPN[3]**（9位，对应第二级页表/根页表索引）[4-6]。
    *   `[29:21]`：**VPN[7]**（9位，对应第一级页表索引）[4-6]。
    *   `[20:12]`：**VPN**（9位，对应第零级页表索引）[4-6]。
    *   `[11:0]`：**Page Offset**（12位，页内偏移，4KB页大小）[1, 2, 4]。
    *   *注：第 39-63 位必须与第 38 位一致（符号位扩展）[8]。*

*   **页表项 (PTE, 64位)**：存储在物理内存中，每个 8 字节 [1, 4, 9]。
    *   `[53:10]`：**PPN**（44位物理页号）[1, 5, 6]。
    *   `[9:0]`：**标志位** [1, 9]。

### 2. 页表项（PTE）标志位含义
Sv39 的页表项包含多个控制位，用于权限管理和状态记录 [1, 2, 9]：
*   **V (Valid)**：有效位。若为 0，访问该项将触发缺页异常 [2, 10, 11]。
*   **R (Readable)** / **W (Writable)** / **X (Executable)**：读、写、执行权限。若三者全为 0，该 PTE 指向下一级页表；否则为叶子节点 [1, 2, 9]。
*   **U (User)**：用户态访问标志。为 1 表示 U 模式可访问 [1, 2, 9]。
*   **G (Global)**：全局映射标志，通常用于操作系统页面，可提高 TLB 性能 [1, 2, 9]。
*   **A (Accessed)**：访问位，记录自上次清除以来页面是否被访问过 [1, 2, 9]。
*   **D (Dirty)**：脏位，记录页面是否被写入过，用于辅助写回策略 [1, 2, 5]。

### 3. 地址转换步骤
地址翻译过程本质上是一个**多级页表遍历**（Page Table Walk）的过程 [4, 6, 12]：

1.  **获取根页表地址**：从 `satp` 寄存器获取根页表（L2）的物理页号，左移 12 位得到物理基地址 [4, 6, 13]。
2.  **L2 索引**：取 `VA[38:30]` 作为索引。页表项地址 = `根基址 + VPN[3] * 8` [4, 14]。读取 PTE2。
3.  **逐级深入**：
    *   检查 PTE2：若为叶子节点（R/W/X 不全为 0），则为 1GB 巨页映射 [6, 15]。
    *   否则，取 `PTE2.PPN` 作为 L1 页表基址，以 `VA[29:21]` 为索引读取 PTE1 [4, 14]。
    *   检查 PTE1：若为叶子节点，则为 2MB 巨页映射 [6, 15]。
    *   否则，取 `PTE1.PPN` 作为 L0 页表基址，以 `VA[20:12]` 为索引读取 PTE0 [4, 14]。
4.  **生成物理地址**：取最终叶子 PTE（如 PTE0）的 `PPN`，与 VA 的 `Page Offset` 拼接 [4, 14, 16]。
    *   **PA = (LeafPTE.PPN << 12) | VA[11:0]** [14]。

### 4. 简化转换数值例
假设 `satp.PPN = 0x80000`（根页表位于 `0x8000_0000`），我们要访问 `VA = 0x8_0000_2ABC`：

1.  **解析 VA**：VPN[3]=0x20, VPN[7]=0, VPN=2, Offset=0xABC。
2.  **访问 L2**：地址 `0x8000_0000 + 0x20 * 8 = 0x8000_0100`。读得 PTE2，其 `PPN = 0x81000`。
3.  **访问 L1**：基址 `0x8100_0000`。地址 `0x8100_0000 + 0 * 8 = 0x8100_0000`。读得 PTE1，其 `PPN = 0x82000`。
4.  **访问 L0**：基址 `0x8200_0000`。地址 `0x8200_0000 + 2 * 8 = 0x8200_0010`。读得 PTE0，其 `PPN = 0x90000`（标志位有效且具有 R/W 权限）。
5.  **拼接 PA**：最终物理地址为 `0x90000` 拼接偏移量 `0xABC` = **`0x9_0000_0ABC`**。

在多核处理器的监听协议中，针对写操作引发的数据一致性维护，主要有以下两种策略：

### 1. 写更新协议 (Write Update / Broadcast)
*   **原理**：当一个核心修改 Cache 中的数据时，立即在总线上广播新数据，强制所有持有该数据副本的其他 Cache 同步更新 [1-3]。
*   **带宽开销**：**很高**。对同一数据的多次连续写操作，每一次都会触发总线广播，极易造成“广播风暴”，占用大量带宽 [3, 4]。
*   **典型应用**：适用于读操作频率远高于写操作，或对读取延迟要求极高的特定专用加速器及数据流明确的计算模式 [3]。
*   **时序场景举例** [1]：
    1.  **初始状态**：P1 和 P2 均持有数据 X=0。
    2.  **P1 执行写操作**：将 X 改为 1。
    3.  **总线动作**：P1 发起“写广播”。
    4.  **最终结果**：P2 监听到广播，立即将本地 Cache 中的 X 更新为 1。

### 2. 写作废协议 (Write Invalidate) —— 主流策略
*   **原理**：在写入数据前，处理器必须先获取独占权，在总线上发送一个“作废”信号，令其他所有 Cache 中的该副本失效（标记为 Invalid）。之后处理器可自由修改本地副本 [1, 3, 5]。
*   **带宽开销**：**较低**。对于连续写操作，仅在第一次写（获取独占权时）产生一次总线作废消息，后续写操作无需产生总线事务 [3, 4]。
*   **典型应用**：绝大多数现代通用多核 CPU 系统的首选（如采用 MSI/MESI 协议的处理器），因为它更节省带宽且更节能 [3, 4]。
*   **时序场景举例** [1, 6]：
    1.  **初始状态**：P1 和 P2 均持有数据 X=0。
    2.  **P1 执行写操作**：计划将 X 改为 1。
    3.  **总线动作**：P1 发起“作废 X”。
    4.  **最终结果**：P2 收到消息，将本地 Cache 中的 X 标记为 **Invalid**；P1 获得独占权并将本地 X 更新为 1。

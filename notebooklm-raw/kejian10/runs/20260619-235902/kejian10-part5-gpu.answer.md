根据**课件 10（向量体系结构/线程级并行）**及 **Week 14-15 课程记录**，GPU 与大规模并行的核心知识总结如下：

### 1. GPU (SIMT) 与向量处理器的对比
两者均旨在挖掘**数据级并行 (DLP)**，但在实现路径上有显著差异：
*   **执行模型**：向量机采用 **SIMD** 模型，指令显式操作向量寄存器；GPU 采用 **SIMT（单指令多线程）** 模型，程序员编写标量线程代码，由硬件将 32 个线程动态组合成一个 **Warp** 执行 [1-3]。
*   **访存与延迟隐藏**：向量机通过**深度流水线**分摊访存延迟（一次支付，后续流水）；GPU 则依靠**大规模多线程**，当一组线程等待内存时，硬件执行零开销切换到另一组线程，以此掩盖延迟 [4-6]。
*   **寄存器组织**：向量机寄存器堆持有完整向量（如 RV64V 为 32 个寄存器）；GPU 将向量元素散布在各车道（Lane）的寄存器中，且寄存器数量远多于向量机，以支持成千上万的活跃线程 [7-9]。
*   **控制流处理**：两者都使用掩码（Mask）处理条件分支，但 GPU 硬件内置 **分支同步栈 (SIMT-Stack)**，能自动管理分支发散与汇聚，对程序员更透明但可能带来性能损耗 [10-12]。

### 2. 现代 AI 集群互连背景
随着大模型参数规模突破 TB 级，单卡算力已无法满足需求，高速互连成为核心：
*   **NVLink 技术**：针对 PCIe 带宽瓶颈（如训练时需数百 GB 吞吐），英伟达推出 NVLink 实现 GPU 间高速直连，支持原子操作和 GPU 直接访问 CPU 内存 [13-15]。
*   **拓扑结构**：大规模集群不再仅使用简单总线，而是采用 **胖树 (Fat Tree)**、**多维环网 (Torus)** 或 **超立方体 (Hypercube)** 等拓扑，以平衡连接成本与等分带宽 [16-18]。
*   **存储空间抽象**：通过 **统一虚拟地址 (UVA)** 和 **统一内存 (Unified Memory)** 技术，多个 GPU 和 CPU 可以共享同一逻辑地址空间，极大简化了多卡编程复杂度 [13, 19, 20]。

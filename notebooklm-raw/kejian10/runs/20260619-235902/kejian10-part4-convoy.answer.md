以下是根据**课件 10（向量体系结构）**及复习笔记整理的向量执行优化重点：

### 1. 护航指令组 (Convoy) 与 钟鸣 (Chime)
*   **护航指令组 (Convoy)**：指一组可以同时在流水线中执行而没有**结构冒险**的向量指令序列 [1, 2]。
    *   若指令间存在写后读 (RAW) 相关，通常需分在不同的 Convoy 中，除非硬件支持**链接 (Chaining)** 技术 [3, 4]。
*   **钟鸣 (Chime)**：执行一个护航指令组所需的时间单位 [1, 5]。
    *   度量方式：对于长度为 $n$ 的向量，执行 $m$ 个护航指令组大约需要 $m$ 次 Chime，即 $m \times n$ 个时钟周期（忽略启动延迟）[5-7]。

### 2. 条带挖掘 (Strip-mining)
当待处理的向量长度 $n$ 超过硬件最大向量长度 ($MVL$) 时，编译器会将循环拆分为多个大小 $\le MVL$ 的分段 [8-10]。
*   **实现机制**：利用**向量长度寄存器 ($vl$)** 控制每次操作的元素个数 [10, 11]。
*   **RISC-V 示例**：使用 `setvl` 指令计算 $vl = \min(mvl, n)$，并在循环中递减 $n$ [10, 12, 13]。

### 3. 向量遮罩 (Mask) 处理 IF 语句
向量处理器不使用分支预测，而是通过**向量遮罩寄存器**实现条件执行 [14, 15]。
*   **原理**：掩码寄存器的每一位对应向量中的一个元素。若该位为 1，则执行运算并写回结果；若为 0，则禁止写回 [15-17]。
*   **优点**：将控制流转换为数据流，避免了标量处理器中分支预测失败带来的流水线冲刷处罚 [14, 18]。

### 4. 数值计算例题
**题目**：假设向量长度 $n=200$，最大向量长度 $MVL=64$，硬件具有 **4 条车道 (Lanes)**。计算该向量任务所需的条带挖掘次数及总执行周期（仅考虑 Chime 部分）。

**逐步手算过程**：
1.  **计算条带挖掘次数 (Strip iterations)**：
    *   次数 = $\lceil n / MVL \rceil$
    *   $\lceil 200 / 64 \rceil = \lceil 3.125 \rceil = 4$ 次 [19]。
    *   *说明：前 3 次处理 64 个元素，最后 1 次处理 $200 - 64 \times 3 = 8$ 个元素。*

2.  **计算单次 Chime 的执行周期**：
    *   在多车道机器下，1 次 Chime 处理 $MVL$ 个元素的时间 = $MVL / \text{车道数}$ [20, 21]。
    *   满载时（前 3 次）：$64 / 4 = 16$ 周期。
    *   末尾剩余时（第 4 次）：$8 / 4 = 2$ 周期。

3.  **计算总周期数**（假设代码序列为 3 个 Convoy [22]）：
    *   总周期 = $\sum (\text{Convoys} \times \lceil \text{当前分段长度} / \text{车道数} \rceil)$
    *   前 3 次循环：$3 \times 16 \times 3 = 144$ 周期。
    *   第 4 次循环：$3 \times 2 = 6$ 周期。
    *   **总执行周期 $\approx 150$ 周期**。

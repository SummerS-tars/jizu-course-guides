根据课件 10（向量体系结构）及相关复习笔记，以下是 5 组最易混淆概念的对比说明：

### 向量体系结构核心概念对比表

| 概念组 | 维度 | 概念 A | 概念 B |
| :--- | :--- | :--- | :--- |
| **1. 并行层次** | **定义** | **ILP (指令级并行)**：通过流水线、多发射或乱序执行让单条指令流内部的操作重叠执行 [1-3]。 | **DLP (数据级并行)**：对集合或数组中的多个元素同时执行相同的操作 [3-5]。 |
| | **开发方式** | 编译器和硬件配合，通常对程序员透明 [6]。 | 显式并行，通常需要重新组织应用结构 [3, 6]。 |
| **2. 执行模型** | **指令流** | **SIMD (单指令多数据)**：单条向量指令指定多个数据输入，在固定宽度的向量寄存器上操作 [7, 8]。 | **SIMT (单指令多线程)**：GPU模型，将多个标量线程指令动态组织成线程束（Warp）执行 [7, 9, 10]。 |
| | **灵活性** | 向量宽度显式，数据间通常无依赖 [11]。 | 每个线程可独立执行路径（尽管分支发散会损耗性能） [7, 12, 13]。 |
| **3. 访存模式** | **内存布局** | **连续访存 (Unit-stride)**：访问内存中地址连续存储的元素（步幅 = 1） [14-16]。 | **步幅访存 (Strided)**：访问内存中以固定间隔分布的元素（步幅 > 1），用于多维数组 [14, 17, 18]。 |
| | **硬件开销** | 提供最高的内存带宽利用率 [14]。 | 容易导致存储体冲突（Bank Conflict），增加延迟 [17, 19]。 |
| **4. 性能度量** | **衡量对象** | **Chime (钟鸣)**：执行一个护航指令组（Convoy）所需的时间单位，用于粗略估算向量耗时 [20, 21]。 | **CPI (每条指令时钟周期)**：衡量标量处理器效率或作为综合平均值的基本性能参数 [22-24]。 |
| | **时间计算** | 向量长度为 $n$、钟鸣数为 $m$ 时，执行时间约 $m \times n$ 个时钟周期 [20, 21]。 | 执行时间 = 指令数 $\times$ CPI $\times$ 时钟周期时间 [25-27]。 |
| **5. 处理机制** | **实现层次** | **Strip-mining (分段开采)**：软件/编译器技术，将超过硬件限制的长向量拆分成 $\le$ MVL 的片段循环处理 [28-30]。 | **向量长度 (VL)**：硬件寄存器，用于控制当前特定向量操作实际处理的元素数量 [29, 31, 32]。 |
| | **关联关系** | 分段开采循环的最后一步通过 `setvl` 指令将剩余长度写入 VL 寄存器 [30, 33]。 | VL 的值不能超过硬件支持的最大向量长度 (MVL) [29, 32]。 |

**(来源：课件10-向量体系结构 / Week15-16 复习笔记)**

根据课件 10 (数据级并行) 及 Week 15-16 复习资料，向量执行优化的核心考点如下：

### 1. 护航指令组 (Convoy) 与 钟鸣 (Chime)
*   **护航指令组 (Convoy)**：指一组可以同时在流水线中重叠执行而没有**结构冒险**的向量指令集合 [1, 2]。
*   **链接技术 (Chaining)**：允许具有先写后读 (RAW) 相关的指令进入同一个护航组，通过流水线定向提高并行度 [2, 3]。
*   **钟鸣 (Chime)**：执行一个护航指令组所需的时间单位 [1]。
    *   对于长度为 $n$ 的向量，执行 $m$ 个护航组大约需要 $m$ 次钟鸣，即 $m \times n$ 个时钟周期 [2, 4]。

### 2. 条带挖掘 (Strip-mining)
当程序向量长度 $n$ 超过硬件最大向量长度 (MVL) 时，需要通过循环将向量切片处理 [5, 6]：
1.  **第一步**：处理不满一个 MVL 的余数部分，长度为 `n % MVL` [6]。
2.  **第二步**：后续循环以 MVL 为步长处理剩余部分 [7]。
3.  **计算公式**：总启动次数 = $\lceil n / MVL \rceil$ [7]。

### 3. 向量遮罩 (Mask) 处理 IF 语句
向量处理器通过**向量遮罩寄存器**实现条件执行，而非分支跳转 [8, 9]：
*   **原理**：谓词指令比较各元素，在 Mask 寄存器中存入布尔值（1 执行，0 屏蔽）[8, 10]。
*   **优势**：将控制流转换为数据流，避免了标量流水线中的分支预测失败惩罚 [9]。

### 4. 数值计算例题
**题目：** 计算向量运算 $A = B \times s$，已知向量长度 $n = 200$，MVL = 64，车道数 (Lanes) = 4。假设该操作包含 1 个护航组。

**逐步手算：**
1.  **计算条带挖掘次数：**
    *   $\text{次次数} = \lceil n / MVL \rceil = \lceil 200 / 64 \rceil = 4$ 次循环 [7]。
2.  **计算单次钟鸣的时钟周期 (忽略启动开销)：**
    *   在一个车道下，执行一次长度为 $n$ 的操作需要 $n$ 个周期。
    *   在 4 车道下，每个周期处理 4 个元素，执行时间 = $n / \text{Lanes} = 200 / 4 = 50$ 个周期 [11, 12]。
3.  **总执行时间 (Chime 视角)：**
    *   该序列有 1 个护航组 ($m=1$)。
    *   不考虑循环开销时，总时间 $\approx 1 \times (n / \text{Lanes}) = 50$ 个周期。
    *   若考虑条带挖掘开销，则总周期 = $4 \times (\text{循环控制开销} + \text{启动延时}) + 50$ [7]。

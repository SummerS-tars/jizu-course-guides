### 1. 一致性定义与实现方法

*   **存储一致性 (Cache Coherence) 定义**：确保对某个数据项的任何读操作都能得到其最新写入的值 [1]。系统需满足：
    *   **写传播 (Write Propagation)**：某个处理器对数据的修改必须传播到其他处理器的副本中 [2]。
    *   **写串行化 (Write Serialization)**：对同一单元的写操作在所有处理器看来顺序必须相同 [3, 4]。
*   **实现方案对比**：
    *   **监听法 (Snoopy)**：每个 Cache 控制器监听共享总线上的所有事务。实现简单但依赖广播，总线带宽是瓶颈，适合小规模多核 [5-7]。
    *   **目录法 (Directory)**：通过集中的或分布式的目录记录各块的共享状态，避免广播，可扩展性强，适合大规模分布式存储系统 [2, 8, 9]。

### 2. 协议演进动机：从 MSI 到 MOESI

*   **MSI → MESI (引入 Exclusive 状态)**：
    *   **动机**：在 MSI 中，若处理器独占读取一个块（进入 S 态），后续写入仍需发总线信号使无效，产生多余开销 [10, 11]。
    *   **E 态意义**：表示数据独占且干净（与主存一致）。从 E 态转为 M 态（写操作）无需总线事务，大大减少了单核私有数据的写开销 [10, 12]。
*   **MESI → MOESI (引入 Owned 状态)**：
    *   **动机**：MESI 在数据从 M 态共享时必须先写回主存，延迟高 [13]。
    *   **O 态意义**：允许 Cache 直接将脏数据传给其他 Cache (Cache-to-Cache Sharing)，且无需立即写回内存。O 态节点作为“拥有者”负责后续数据提供 [14-16]。

### 3. MESI 四状态含义与迁移

| 状态 | 描述 | 内存数据是否最新 |
| :--- | :--- | :--- |
| **M (Modified)** | 已修改，块独占，且为唯一最新副本 [10, 11] | 否 |
| **E (Exclusive)** | 独占且干净，无其他副本 [10, 11] | 是 |
| **S (Shared)** | 可能有多个副本 [11, 17] | 是 |
| **I (Invalid)** | 块无效 [11, 17] | - |

**核心迁移条件**：
*   **本地读命中**：M/E/S 态均不产生总线动作 [18]。
*   **本地写命中**：E 态直接转 M；S 态需发总线使无效信号 (BusUpgr/BusRdX) 转 M [18]。
*   **监听总线读 (BusRd)**：若本地为 M，需将数据 **Flush** 到总线，并降级为 S [19]。
*   **监听总线写 (BusRdX)**：作废本地副本，若为 M 则需 Flush 并在之后转为 I [19]。

### 4. 数值模拟示例（假设初始全为 I）

假设地址 A 初始值为 5，Cache 采用 MSI/MESI 协议：

| 步骤 | 动作 | P1 状态 | P2 状态 | 总线事务 | 数据来源 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | **P1 读 A** | **E** | I | BusRd | 内存 (值为 5) |
| 2 | **P2 读 A** | **S** | **S** | BusRd | 内存/P1 (检测到 Shared 信号) |
| 3 | **P1 写 A=10** | **M** | **I** | BusRdX (使无效) | 本地修改 |
| 4 | **P2 读 A** | **S** | **S** | BusRd + **Flush** | P1 提供 (值为 10)，内存更新 |

*注：步骤 4 中 P1 检测到 BusRd，将 A=10 写回内存并传给 P2，两者均变 S [20-22]。*

### 5. 真共享 vs 伪共享 (False Sharing)

*   **真共享缺失**：不同处理器读写**同一个变量**导致的一致性缺失。这是必要的通信 [23, 24]。
*   **伪共享 (False Sharing)**：
    *   **诊断**：不同处理器访问同一 Cache 行内的**不同变量**（如变量 X 和 Y 在同一行）。P1 写 X 导致 P2 的行失效，P2 读 Y 发生缺失，但实际上 Y 没变 [23-25]。
    *   **特征**：缺失率随 CPU 数量增加而上升，但变量间无逻辑关联。
*   **避免方法**：
    *   **数据填充 (Padding)**：在变量间插入空白字节，使其分布在不同 Cache 行 [26, 27]。
    *   **编译器优化**：通过对齐指令或调整结构体布局，让共享频繁的变量物理隔离 [26, 27]。

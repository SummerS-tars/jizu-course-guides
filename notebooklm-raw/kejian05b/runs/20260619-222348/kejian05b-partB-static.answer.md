根据**课件 5b**的内容，关于静态调度的讲解如下：

### 1. 编译器指令调度的基本思想
编译器通过**重新排列指令顺序**，使具有数据相关的指令之间保持足够的距离（即流水线延迟），从而利用不相关的指令填充原本会产生的“气泡”或停顿（Stall） [1, 2]。

### 2. 循环展开 (Loop Unrolling) 如何暴露 ILP
循环展开通过将循环体复制多次，并相应调整循环结束条件，产生以下效果：
*   **增加基本块大小**：提供了更多的指令供编译器调度 [3]。
*   **降低开销**：减少了分支指令和指针更新指令在总执行指令中的比例 [4]。
*   **创造调度空间**：允许跨迭代移动指令（例如在等待第一个迭代的载入结果时，执行第二个迭代的载入），从而掩盖高延迟操作 [5, 6]。

### 3. 静态寄存器重命名消除 WAR/WAW
在循环展开后，如果仍使用相同的寄存器，会引入**名称相关**（反相关 WAR 和输出相关 WAW），阻碍指令的自由移动 [7, 8]。
*   **方法**：编译器为展开后的不同迭代分配**不同的物理寄存器**。
*   **结果**：消除了“假相关”，使得原本由于寄存器冲突而必须先后执行的指令可以重叠执行 [9, 10]。

### 简单循环展开前后对比示例 [课件5b]

假设执行 `x[i] = x[i] + s`，浮点加法延迟为 3 周期，载入到使用的延迟为 1 周期。

**展开前（低 ILP，存在大量停顿）：**
```mips
Loop: L.D    F0, 0(R1)     ; 载入 x[i]
      stall                ; 等待数据就绪
      ADD.D  F4, F0, F2    ; x[i] + s
      stall                ; 等待加法完成
      stall                
      S.D    F4, 0(R1)     ; 存储结果
      DADDIU R1, R1, #-8   ; 指针递减
      BNE    R1, R2, Loop  ; 分支跳转
```

**展开 4 次并进行调度与重命名（高 ILP，无停顿）：**
```mips
Loop: L.D    F0, 0(R1)     ; 第1次载入
      L.D    F6, -8(R1)    ; 第2次载入（重命名为F6）
      L.D    F10, -16(R1)  ; 第3次载入（重命名为F10）
      L.D    F14, -24(R1)  ; 第4次载入（重命名为F14）
      ADD.D  F4, F0, F2    ; 第1次加法
      ADD.D  F8, F6, F2    ; 第2次加法（重命名为F8）
      ADD.D  F12, F10, F2  ; 第3次加法...
      ADD.D  F16, F14, F2  ; 第4次加法...
      S.D    F4, 0(R1)     ; 第1次存储
      S.D    F8, -8(R1)    ; 第2次存储
      S.D    F12, -16(R1)  ; 第3次存储
      S.D    F16, -24(R1)  ; 第4次存储
      DADDIU R1, R1, #-32  ; 指针一次性更新
      BNE    R1, R2, Loop  ; 分支开销减至 1/4
```
（注：展开后的指令交错执行，消除了等待延迟 [11, 12]。）

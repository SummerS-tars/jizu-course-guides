### 1. Tomasulo 算法动机：动态寄存器换名
Tomasulo 算法的核心动机是通过**硬件动态重命名**消除“假相关” [1, 2]。
*   **消除冲突**：通过保留站（Reservation Station）替代架构寄存器，彻底消除 **WAR（写后读）** 和 **WAW（写后写）** 冲突 [3, 4]。
*   **乱序执行**：允许指令在操作数就绪时立即执行，而不必按程序顺序，从而减少 **RAW（真相关）** 导致的停顿 [3, 5]。
*   **隐藏延迟**：能够跨越分支进行调度，并通过乱序执行隐藏访存和浮点运算的长延迟 [1, 6]。

### 2. 保留站 7 字段含义
保留站是分布式的缓冲区，每个字段记录了指令执行所需的关键信息 [7, 8]：
1.  **Busy**：为“Yes”表示该保留站已被占用 [7, 9]。
2.  **Op**：指令的操作码（如 ADD, MUL, DIV） [7, 10]。
3.  **Vj, Vk**：源操作数的值。只有当对应的 Q 字段为 0 时，值才有效 [7, 10]。
4.  **Qj, Qk**：产生源操作数的保留站编号。若为 0，表示操作数已就绪并存储在 V 字段中 [7, 10]。
5.  **A**：存储访存指令的地址信息。初始存偏移量，计算后存有效地址 [7, 10]。

### 3. CDB 广播、冲突与仲裁
*   **CDB 广播机制**：所有功能部件的计算结果都通过 **公共数据总线（CDB）** 播送。所有等待该结果的保留站和寄存器状态表项会同时捕获该值 [11, 12]。
*   **结构冲突**：若某类指令所需的保留站已满，指令将无法**发射（Issue）**，必须在指令队列中等待 [7, 13]。
*   **CDB 竞争仲裁**：当多个功能部件同时完成计算并请求使用 CDB 时，需要仲裁逻辑决定优先级（如先到先得），未获准的部件需推迟写回 [13, 14]。

### 4. 完整时序表数值例（期末典型题型）
**给定条件**：
*   **延迟**：Load: 1, Add: 2, Mul: 6, Div: 12。
*   **假设**：每个周期发射 1 条指令；寄存器写前半周期，读后半周期 [15, 16]。

#### 指令序列：
1. `L.D F6, 32(R2)`
2. `L.D F2, 44(R3)`
3. `MUL.D F0, F2, F4`
4. `SUB.D F8, F2, F6`
5. `DIV.D F10, F0, F6`
6. `ADD.D F6, F8, F2`

#### 逐步执行过程表：
| 指令 | 发射 (Issue) | 执行 (Execute) | 写回 (Write Result) | 备注 |
| :--- | :---: | :---: | :---: | :--- |
| L.D F6, 32(R2) | 1 | 2 | 3 | 周期 3 通过 CDB 广播 F6 的值 |
| L.D F2, 44(R3) | 2 | 3 | 4 | 周期 4 通过 CDB 广播 F2 的值 |
| MUL.D F0, F2, F4 | 3 | 5-10 | 11 | 等待周期 4 的 F2 结果后开始执行 |
| SUB.D F8, F2, F6 | 4 | 5-6 | 7 | 等待周期 3(F6) 和 4(F2) 结果 |
| DIV.D F10, F0, F6 | 5 | 12-23 | 24 | 等待周期 11 的 F0 结果 |
| ADD.D F6, F8, F2 | 6 | 8-9 | 10 | 等待周期 7 的 F8 结果 |

#### 周期 4 结束时的状态快照：
**保留站状态表 [17, 18]**：
| 名称 | Busy | Op | Vj | Vk | Qj | Qk | A |
| :--- | :---: | :---: | :--- | :--- | :---: | :---: | :--- |
| Load1 | No | | | | | | |
| Load2 | No | | | | | | |
| Add1 | Yes | SUB | | Mem[32+Regs[R2]] | **Load2** | 0 | |
| Mult1 | Yes | MUL | | Reg[F4] | **Load2** | 0 | |

**寄存器状态表 (Qi) [18, 19]**：
| 字段 | F0 | F2 | F4 | F6 | F8 | F10 | ... |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **Qi** | Mult1 | 0 | 0 | 0 | Add1 | 0 | |

*注：周期 4 结束时，Load1 已写回并清除 Qi 标志；Load2 正在写回，此时 Add1 和 Mult1 的 Qj 仍指向 Load2。*

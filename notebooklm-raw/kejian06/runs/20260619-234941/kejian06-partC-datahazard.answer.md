根据**课件 06**（指令流水线）、**Week 8** 笔记及 **Lab1 报告**的要求，为你梳理数据冒险的处理机制：

### 1. 数据冒险类型 [1, 2]
*   **RAW (写后读)**：最常见，后续指令需要读取前序指令尚未写回的结果（真相关）[2, 3]。
*   **WAR (读后写)**：后续指令在较早指令读取前先写入寄存器（反相关）。在 5 级顺序流水线中通常不发生 [3, 4]。
*   **WAW (写后写)**：后续指令在较早指令写入前先完成写入（输出相关）[4, 5]。
*   **Load-Use 冒险**：特指 `lw` 指令后紧跟使用该寄存器的指令。由于数据在 MEM 级结束才就绪，存在“时间错位”，无法仅靠转发解决 [6, 7]。

### 2. 转发 (Forwarding) 机制 [8-10]
转发通过增加 MUX 旁路，将结果从流水段寄存器直接送回 ALU 输入端 [8, 10]。

**检测条件：**
*   **EX 级转发**（来自上一条）：`EX/MEM.RegWrite` 且 `Rd != 0` 且 `EX/MEM.Rd == ID/EX.Rs (或 Rt)` [7, 8]。
*   **MEM 级转发**（来自上上条）：`MEM/WB.RegWrite` 且 `Rd != 0` 且 `MEM/WB.Rd == ID/EX.Rs (或 Rt)` [7, 8]。
*   **优先级**：EX 级转发优先级高于 MEM 级，因为 EX 级的数据更“新” [7, 9]。

### 3. 阻塞 (Stall) 与气泡 [7, 11, 12]
当转发无法解决（如 Load-Use）时，必须阻塞流水线 [6, 7]。
*   **实现方法**：保持 PC 和 IF/ID 寄存器值不变（指令重新译码）；同时将 ID/EX 段控制信号清零，产生一个 **NOP（气泡）** [7, 11]。

### 4. 数值例：流水线时空图 [1, 6, 7, 12]
假设指令序列如下：
1. `add r1, r2, r3`
2. `sub r4, r1, r5` (依赖 r1)
3. `lw  r2, 0(r6)`
4. `and r7, r2, r8` (Load-Use 依赖 r2)
5. `or  r9, r7, r10` (依赖 r7)

| 指令 | C1 | C2 | C3 | C4 | C5 | C6 | C7 | C8 | C9 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 1. add | IF | ID | EX | MEM | WB | | | | | 产生 r1 [13] |
| 2. sub | | IF | ID | **EX** | MEM | WB | | | | **转发**来自 1 的 r1 [14] |
| 3. lw | | | IF | ID | EX | MEM | WB | | | 产生 r2 [13] |
| 4. and | | | | IF | **ID** | **stall** | **EX** | MEM | WB | **阻塞** 1 周期 [7, 12] |
| 5. or | | | | | IF | **stall** | ID | **EX** | MEM | **转发**来自 4 的 r7 [7] |

*   **转发发生在 C4**：`add` 的结果从 `EX/MEM` 转发给 `sub` [7, 14]。
*   **阻塞发生在 C6**：由于 `lw` 的数据在 C6 结束才就绪，`and` 必须在 ID 级等待一拍，插入气泡 [7, 12]。

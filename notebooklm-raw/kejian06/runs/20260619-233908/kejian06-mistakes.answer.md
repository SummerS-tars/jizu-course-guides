根据**课件 06 指令流水线**及相关课程资料，为您对比总结了 5 组最易混淆的核心概念：

### 1. 三类流水线冒险对比
| 概念 | 产生原因 | 典型解决办法 |
| :--- | :--- | :--- |
| **结构冒险** | 硬件资源冲突（如多条指令同时访存） | 物理分离指令/数据存储器、寄存器读写分离 [1-3] |
| **数据冒险** | 指令间存在数据依赖（RAW/WAR/WAW） | 转发（Forwarding）、硬件阻塞（Stall） [1, 4, 5] |
| **控制冒险** | 分支/跳转指令改变执行流导致取错指令 | 分支预测、延迟分支、流水线冲刷（Flush） [1, 6, 7] |

### 2. 数据冒险处理：转发 vs 阻塞
| 概念 | 原理 | 性能影响 |
| :--- | :--- | :--- |
| **转发 (Forwarding)** | 直接从段间寄存器（如 EX/MEM）取回计算结果送入 ALU 输入端 [4, 8, 9] | **无损失**，指令连续执行 |
| **阻塞 (Stalling)** | 硬件停止取指和译码阶段，并向后级流水线插入“气泡” [6, 10, 11] | **有损失**，CPI 增加（如 Load-Use 冒险必须阻塞） [12, 13] |

### 3. 流水线控制：Flush vs Stall
| 概念 | 适用场景 | 实现动作 |
| :--- | :--- | :--- |
| **Flush (冲刷)** | 分支预测失败或发生异常 [14, 15] | 将流水线中已有的错误指令清零（变成 NOP），**抛弃**已取指令 [15-17] |
| **Stall (阻塞)** | 等待数据就绪（如 Load-Use） [12, 18] | 保持 PC 和当前流水段寄存器不变，**暂停**后续指令进入 [11, 18, 19] |

### 4. 性能评价：吞吐率 vs 加速比
| 概念 | 定义 | 计算重点 |
| :--- | :--- | :--- |
| **吞吐率 (Throughput)** | 单位时间内流水线完成的指令数量 [20-22] | 衡量流水线**并行处理能力** [23, 24] |
| **加速比 (Speedup)** | 同一任务在流水线与非流水线下的执行时间比 [20, 21, 25] | 衡量**性能提升的倍数** [26, 27] |

### 5. 寄存器分类：段间寄存器 vs 通用寄存器
| 概念 | 可见性 | 存储内容 |
| :--- | :--- | :--- |
| **段间寄存器** | 用户不可见（透明） [28-30] | 存放指令各阶段的中间结果及随指令流动的**控制信号** [29, 31, 32] |
| **通用寄存器 (GPR)** | 用户可见（汇编可访问） [28, 33] | 存放程序变量、数据操作数及地址 [28, 34] |

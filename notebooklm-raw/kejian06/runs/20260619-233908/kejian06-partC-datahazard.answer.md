根据**课件 06**及相关课程笔记，数据冒险（Data Hazard）是流水线设计中处理指令间数据依赖的核心挑战。以下是针对您要求的详细讲解：

### 1. 数据冒险分类
根据指令读写访问的顺序，数据冒险分为三类（主要发生于指令 $i$ 在 $j$ 之前进入流水线时）：

*   **RAW (Read After Write，写后读)**：指令 $j$ 试图在指令 $i$ 写入寄存器前读取该值，导致读取错误。这是最常见的真数据相关 [1-3]。
*   **WAR (Read After Write，读后写)**：指令 $j$ 在指令 $i$ 读取前写入。在经典 5 级**顺序**流水线中不会发生，但在乱序执行时可能出现 [1, 3, 4]。
*   **WAW (Write After Write，写后写)**：指令 $j$ 在指令 $i$ 写入前提前写入，导致最终结果被指令 $i$ 覆盖。同样仅多见于乱序流水线 [1, 3, 4]。
*   **Load-Use 冒险**：特指 Load 指令后紧跟需要该数据的指令。由于数据需在 MEM 段结束才就绪，此时后续指令已进入 EX 段，产生“时间错位”，仅靠转发无法消除，**必须阻塞 1 个周期** [5-7]。

---

### 2. 转发 (Forwarding) 检测与路径
转发通过在硬件中增加多路选择器（MUX），将中间结果直接送往需要的执行部件，无需等待 WB 阶段 [8, 9]。

**检测条件（以 EX 级转发为例）：**
*   **条件 1 (C1a)**：`EX/MEM.RegWrite` 为真，且 `EX/MEM.RegisterRd != 0`，且其等于当前 `ID/EX.RegisterRs` [7, 10]。
*   **优先权**：若 EX 级和 MEM 级转发同时满足，优先选择 **EX 级转发**（即更近的数据更新） [6, 7]。

**旁路路径：**
*   **EX/MEM -> ALU 输入**：转发上一条指令的 ALU 结果 [11]。
*   **MEM/WB -> ALU 输入**：转发上上条指令的 ALU 或访存结果 [11]。

---

### 3. 阻塞 (Stall) 与气泡插入
当转发无法解决冲突（如 Load-Use）时，必须通过硬件阻塞延迟后续指令执行 [6, 12]。

*   **检测点**：处于 ID 段时检测。若 `ID/EX.MemRead == 1`（前序是 Load）且 `ID/EX.RegisterRt` 等于当前指令的源寄存器 [7, 13]。
*   **实现动作**：
    1.  **保持 PC 不变**：防止新指令取指 [7, 14]。
    2.  **保持 IF/ID 寄存器不变**：让当前受阻指令下周期重新译码 [7, 14]。
    3.  **插入气泡 (Bubble)**：将 ID/EX 段寄存器的所有控制信号清零，使其在后续段中等效为 NOP 指令 [7, 14, 15]。

---

### 4. 数值例：流水线时空图
给定以下 5 条指令序列，演示转发与阻塞的协同工作：
1. `add x1, x2, x3`
2. `sub x4, x1, x5` (RAW: x1, 转发自 I1)
3. `lw  x2, 8(x4)` (RAW: x4, 转发自 I2)
4. `and x5, x2, x1` (Load-Use: x2, 需阻塞 1 拍 + 转发自 I3)
5. `or  x6, x5, x2` (RAW: x5, 转发自 I4)

| 周期 (Cycle) | T1 | T2 | T3 | T4 | T5 | T6 | T7 | T8 | T9 | T10 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **I1: add** | IF | ID | EX | M | WB | | | | | |
| **I2: sub** | | IF | ID | **EX\*** | M | WB | | | | |
| **I3: lw** | | | IF | ID | **EX\*** | M | WB | | | |
| **I4: and** | | | | IF | ID | **Stall** | **EX\*** | M | WB | |
| **I5: or** | | | | | IF | **Stall** | ID | **EX\*** | M | WB |

**标注说明：**
*   **EX\***：使用了转发技术，I2 的 x1 来自 I1 的 EX/MEM 段，I3 的 x4 来自 I2 的 EX/MEM 段，I4 的 x2 来自 I3 的 MEM/WB 段（阻塞后转发） [7, 11, 16]。
*   **Stall**：由于 I3 是 Load 且 I4 立即使用其结果 x2，产生 Load-Use 冒险，在 T6 周期通过清空 ID/EX 控制信号插入气泡 [7, 13]。

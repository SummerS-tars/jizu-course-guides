根据**课件 06（Slide 7–19）**及相关课程资料，五级流水线的实现及 Lab 1-2 的对应关系总结如下：

### 1. 五级功能与段间寄存器
流水线通过在各阶段间插入**段间寄存器**（Buffer），将单周期路径切分为独立部分，以实现指令重叠执行 [1, 2]。

*   **IF (取指)**：从指令存储器取指令并计算 `PC+4` [1, 3]。
    *   **IF/ID 寄存器**：保存取出的指令和 `PC+4` [1, 3, 4]。
*   **ID (译码/读寄存器)**：解析指令并从寄存器堆读出操作数 [1, 5]。
    *   **ID/EX 寄存器**：保存 `Reg[Rs]`, `Reg[Rt]`, `Imm`, `Rd/Rt`（目的寄存器号）等 [1, 4, 5]。
*   **EX (执行)**：ALU 进行运算或计算访存地址 [1, 5]。
    *   **EX/MEM 寄存器**：保存 ALU 运算结果、Zero 标志及写存数据 [1, 4, 6]。
*   **MEM (访存)**：读/写数据存储器 [1, 6]。
    *   **MEM/WB 寄存器**：保存存储器读出的数据或 ALU 运算结果 [1, 4, 6]。
*   **WB (写回)**：将最终结果写回目的寄存器 [1, 6]。

**对照 Lab 实现**：Lab 1 的核心就是搭建这四组段间寄存器作为“骨架”，确保指令能在各阶段稳定流动 [7]。

### 2. 控制信号的传递
流水线的控制器逻辑类似于单周期设计，但需要**同步传递** [1, 8, 9]。

*   **集中产生**：所有控制信号在 **ID 阶段**由译码器一次性产生 [1, 10, 11]。
*   **流水传递**：控制信号被保存在段间寄存器中，随指令同步流向后续阶段 [1, 4, 12]。
    *   **EX 级信号**（如 `ALUSrc`, `ALUOp`）：产生后下个周期立即使用 [1, 10]。
    *   **MEM 级信号**（如 `MemWrite`, `Branch`）：需传递 2 个周期 [1, 10, 13]。
    *   **WB 级信号**（如 `RegWrite`, `MemtoReg`）：需传递 3 个周期 [1, 8, 10]。

**对照 Lab 实现**：在 `core_decode.sv` 中补齐控制信号（如 `mem_read`, `wb_sel`），并通过顶层时序逻辑向后传递 [14]。

### 3. 指令数据流（Data Flow）

#### `add rd, rs, rt` 的数据流：
1.  **IF**：取指令 [1, 15]。
2.  **ID**：读寄存器 `rs` 和 `rt` [1, 15]。
3.  **EX**：ALU 进行加法运算 [1, 16]。
4.  **MEM**：空操作（仅将 ALU 结果传给下一级）[1, 16, 17]。
5.  **WB**：将 ALU 结果写回 `rd` [1, 15, 16]。

#### `lw rt, imm(rs)` 的数据流：
1.  **IF**：取指令 [1, 18]。
2.  **ID**：读寄存器 `rs`，对立即数 `imm` 进行符号扩展 [1, 5]。
3.  **EX**：ALU 计算有效地址（`rs + imm`）[1, 5]。
4.  **MEM**：根据计算的地址从数据存储器读数据 [1, 6, 19]。
5.  **WB**：将内存读出的数据写回 `rt` [1, 6]。

**对照 Lab 实现**：Lab 1 实现了算术指令流，Lab 2 重点在于补齐 MEM 阶段的 `dreq` 访存通路，并处理 `load-use` 引起的阻塞 [7, 20, 21]。

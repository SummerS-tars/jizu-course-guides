根据**课件 09 MIPS32 流水线处理器**及相关实验资料，为你讲解五级流水结构的核心内容：

### 1. 单周期 vs 流水线时序对比
*   **单周期（Single-Cycle）：** 每条指令都在一个极长的时钟周期内完成（例如 900ps），时钟周期由最慢的指令（如 `lw`）决定 [1, 2]。
*   **流水线（Pipelined）：** 将指令执行拆分为 5 个较短的阶段，时钟周期缩短为最慢阶段的时间（例如 200ps） [1, 3]。
*   **性能提升：** 虽然流水线并不缩短单条指令的执行时间（ latency 甚至因寄存器开销略有增加），但显著提高了**指令吞吐率**。理想情况下，主频可提升约 5 倍 [1, 4, 5]。

### 2. IF/ID/EX/MEM/WB 阶段与 RISC-V Lab1 对照
课件 [1, 6] 定义的五级结构与 **Lab1 报告** [7] 的设计目标完全一致：
*   **IF (Instruction Fetch, 取指)：** 从指令存储器取出指令，更新 PC [1, 7]。
*   **ID (Instruction Decode, 译码)：** 指令译码并读取通用寄存器操作数 [1, 7]。
*   **EX (Execute, 执行)：** ALU 进行算术逻辑运算或计算访存地址 [1, 7]。
*   **MEM (Memory, 访存)：** 读写数据存储器（Lab1 阶段主要为算术指令，此级多为占位或透传） [7, 8]。
*   **WB (Write Back, 写回)：** 将运算结果或访存数据写回寄存器堆 [1, 7]。

### 3. 多条指令重叠执行的时空图
*   **重叠机制：** 在时空图中，每一行代表一条指令，每一列代表一个时钟周期。当指令 $I_1$ 进入 ID 阶段时，$I_2$ 即可进入 IF 阶段 [9, 10]。
*   **硬件利用率：** 理想状态下，5 个功能部件（IM、RF、ALU、DM、RF）在每个周期都同时为不同的指令服务 [9, 11]。
*   **结果：** 流水线填满后，每个周期都有一条指令进入并有一条指令完成提交，CPI 趋近于 1 [1, 11, 12]。

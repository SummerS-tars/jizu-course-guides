根据**课件 04 指令系统**及相关课程笔记，以下是 5 组最易混淆的概念对比说明：

### 1. RISC-V 六种指令格式适用场景
| 格式 | 全称 | 典型适用指令 | 特点 |
| :--- | :--- | :--- | :--- |
| **R-type** | Register | `add`, `sub`, `and` | 纯寄存器间运算，含两个源寄存器 [1, 2] |
| **I-type** | Immediate | `addi`, `lw`, `jalr` | 立即数运算、加载指令、寄存器跳转 [1-3] |
| **S-type** | Store | `sw`, `sh`, `sb` | 存储指令，将寄存器值存入内存 [1, 3] |
| **B-type** | Branch | `beq`, `bne`, `blt` | 条件分支，偏移量需乘以 2（16 位对齐） [1, 4] |
| **U-type** | Upper | `lui`, `auipc` | 处理 20 位长立即数，加载到高位 [1, 5] |
| **J-type** | Jump | `jal` | 无条件跳转并链接返回地址 [1, 5] |
*(来源：课件04)*

### 2. ABI 寄存器名 vs 硬件编号
| 维度 | 硬件编号 (Hardware Name) | ABI 寄存器名 (ABI Name) |
| :--- | :--- | :--- |
| **表示** | `x0` - `x31` | `zero`, `ra`, `sp`, `a0-a7`, `s0-s11`, `t0-t6` |
| **可见性** | 硬件解码器唯一识别的标识 [1, 6] | 程序员和编译器使用的助记符 [1, 7] |
| **功能约定** | 物理上除 `x0` 外基本对称 [8] | 规定了调用规范（如 `a0` 传参，`ra` 存返回地址） [1, 9] |
| **典型示例** | `x1` | `ra` (Return Address) [1, 9] |
*(来源：课件04)*

---

### 3. 立即数寻址 vs PC 相对寻址
| 特性 | 立即数寻址 (Immediate) | PC 相对寻址 (PC-relative) |
| :--- | :--- | :--- |
| **地址计算** | 操作数直接包含在指令字中 [10, 11] | 有效地址 = 当前 PC + 偏移量 [11-13] |
| **适用指令** | `addi`, `ori` 等运算类 [11, 14] | `beq`, `bne` (分支)；`jal` (跳转) [11, 15] |
| **核心优势** | 指令执行快，无需额外访存 [10, 16] | 支持位置无关代码 (PIC)，利于程序浮动 [12, 17, 18] |
| **寻址范围** | 受指令中立即数字段位数限制 [10, 16] | 相对于当前指令的前后跳转范围 [15, 19] |
*(来源：课件04 / Week3)*

---

### 4. CISC vs RISC 风格
| 维度 | CISC (复杂指令集) | RISC (精简指令集) |
| :--- | :--- | :--- |
| **指令长度** | 可变长度（如 x86: 1~15 字节） [20, 21] | 固定长度（如 RISC-V: 32 位） [1, 20, 21] |
| **访存方式** | 运算指令可直接访问内存 [22-24] | 仅 `Load/Store` 指令访存，运算仅在寄存器间 [23-25] |
| **寄存器** | 数量较少，多为专用寄存器 [24, 26] | 数量较多（通常 32 个），多为通用寄存器 [24, 26, 27] |
| **控制器** | 微程序控制为主 [22, 24] | 硬布线逻辑（组合逻辑）为主，适合流水线 [24, 28] |
*(来源：课件04)*

---

### 5. ISA vs ABI 界面
| 界面 | 指令集体系结构 (ISA) | 应用程序二进制接口 (ABI) |
| :--- | :--- | :--- |
| **定义层次** | 软件与硬件的交界面 [29-31] | 应用程序与系统（硬件+OS）的界面 [32-34] |
| **涵盖内容** | 指令集、寄存器结构、内存编址、特权级 [29, 31, 35] | 数据类型对齐、寄存器使用约定、函数调用规范 [32, 34] |
| **作用** | 保证软件在不同硬件实现间的二进制兼容 [36, 37] | 保证不同编译器生成的二进制模块能互操作 [34, 38] |
| **感知者** | 系统程序员（写 OS、编译器的人） [29, 39] | 应用程序员、编译器开发者 [34, 39] |
*(来源：课件04 / Week3)*

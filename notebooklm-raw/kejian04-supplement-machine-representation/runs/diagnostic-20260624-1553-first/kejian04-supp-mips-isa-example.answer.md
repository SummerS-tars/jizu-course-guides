课件选择 **MIPS ISA** 作为教学例子，主要基于其作为**典型精简指令集（RISC）**的代表性，能够清晰地展示高级语言、汇编语言与机器语言之间的等价转换过程 [1, 2]。

具体原因组织如下：

### 1. MIPS 是最典型的 RISC 架构
MIPS 指令集设计极其纯粹，最初仅有 58 条整数指令，且专为单发射、顺序流水线实现而设计 [1]。这种**简单性**使其成为初学者理解“机器如何理解程序”的最佳范本 [1, 3]。

### 2. 与第一讲 ISA 设计原则的衔接
在第一讲中提到的 ISA 设计核心原则，在 MIPS 的机器表示中得到了直观体现：
*   **规整性与简单性**：第一讲强调指令系统应易于硬件设计 [4]。MIPS 仅采用 **R、I、J 三种定长指令格式**，操作码长度固定，极大简化了第二讲中提到的取指与译码逻辑 [2, 3]。
*   **Load/Store 型结构**：对应第一讲中对操作数地址指定方式的分类，MIPS 强制要求算术运算指令的操作数必须来自寄存器或立即数，只有专门的访存指令才能访问存储器 [5, 6]。这降低了指令集和硬件的复杂性，方便了流水线实现 [5]。

### 3. 教学与实验的互补
虽然课程原理讲解以 MIPS 为例，但学生实验基于 RISC-V 实现 [3]。由于两者同属 RISC 阵营，设计哲学高度一致（如指令定长、寄存器丰富），使用 MIPS 讲解能让学生在掌握通用原理后，更易于理解实验中 RISC-V 的具体实现 [3, 7]。

通过 MIPS 这个例子，课件成功地将第一讲中抽象的“设计规约”转化为了第二讲中具体的“电路执行过程”与“机器码解码” [2, 8]。

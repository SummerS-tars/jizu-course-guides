在 MIPS 体系结构中，所有指令均为 32 位定长，通过以下三种主要格式映射到机器码字段：

### 1. MIPS 指令格式字段表

| 类型 | 31:26 (6位) | 25:21 (5位) | 20:16 (5位) | 15:11 (5位) | 10:7 (5位) | 5:0 (6位) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **R 型** (寄存器) | **op** | **rs** | **rt** | **rd** | **shamt** | **funct** |
| **I 型** (立即数) | **op** | **rs** | **rt** | **immediate / address (16位)** | | |
| **J 型** (跳转) | **op** | **target address (26位)** | | | | |

（来源：课件04 [1-3]）

### 2. 字段语义说明

*   **op (opcode)**：操作码，指定指令的基本操作类型 [2, 4]。对于所有 R 型指令，该字段恒为 `0` [2, 4]。
*   **rs (Source Register)**：第一源操作数寄存器编号 [2, 4]。
*   **rt (Register Temporary)**：第二源操作数寄存器（R 型）或目标/源寄存器（I 型，如 `lw` 的目标、`sw` 的源） [2, 4]。
*   **rd (Destination Register)**：目标操作数寄存器，用于存放 R 型指令的运算结果 [2, 4]。
*   **shamt (shift amount)**：移位量，仅用于移位指令（如 `sll`），指定移动的位数 [2, 4]。
*   **funct (function)**：功能码，在 `op` 为 0 时，配合 `op` 字段精确指定具体的算术或逻辑操作（如 `add`、`sub`） [2, 4]。
*   **immediate / address**：16 位立即数、数据地址或分支指令的偏移量 [2, 5]。
*   **target address**：26 位跳转目标地址，用于计算最终的 PC 值 [5, 6]。

（来源：课件04 / Week3 记录 [2, 4-6]）

### 3. 逐步编码示例：`add $t0, $s1, $s2`

按照 R 型指令格式进行编码：
1.  **确定操作码和功能码**：查表得 `add` 的 `op = 0` (000000)，`funct = 0x20` (100000) [7]。
2.  **映射寄存器编号**：寄存器 `$s1` 编号为 17 (10001)，`$s2` 编号为 18 (10010)，目标寄存器 `$t0` 编号为 8 (01000) [8, 9]。
3.  **确定移位量**：非移位指令，`shamt = 0` (00000) [9]。
4.  **拼接字段（二进制）**：
    `000000 (op)` | `10001 (rs)` | `10010 (rt)` | `01000 (rd)` | `00000 (shamt)` | `100000 (funct)` [9]。
5.  **转换十六进制**：`0x02324020` [10]。

（来源：课件04 / Week3 记录 [9, 10]）

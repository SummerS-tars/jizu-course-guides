### Cache 映射原理与地址划分

在存储层次结构中，主存块映射到 Cache 的方式决定了硬件如何查找数据及主存地址的逻辑划分。

#### 1. 直接映射 (Direct Mapping)
*   **原理**：每个主存块只能映射到 Cache 中唯一固定的行 [1, 2]。
*   **映射公式**：Cache 行号 = 主存块号 $\bmod$ Cache 总行数 [3, 4]。
*   **地址划分**：
    *   **标记 (Tag)**：主存地址的高位，用于区分映射到同一行的不同主存块 [2, 5]。
    *   **行索引 (Index)**：中间位，用于直接定位 Cache 中的具体行 [5, 6]。
    *   **块内偏移 (Offset)**：低位，用于在块内定位具体的字节或字 [5, 7]。

#### 2. 组相联映射 (Set-Associative Mapping)
*   **原理**：将 Cache 分成若干组，每组包含 $N$ 行。主存块映射到固定组的任意一行（组间直接映射，组内全相联）[8-10]。
*   **映射公式**：Cache 组号 = 主存块号 $\bmod$ Cache 总组数 [4, 8]。
*   **地址划分**：
    *   **标记 (Tag)**：高位，标识主存组群 [9, 11]。
    *   **组索引 (Index)**：中间位，定位 Cache 组 [11, 12]。
    *   **块内偏移 (Offset)**：低位，块内寻址 [9, 11]。

#### 3. 全相联映射 (Fully Associative Mapping)
*   **原理**：主存块可以装入 Cache 的任何一行，灵活性最高，冲突缺失最少 [1, 4, 9]。
*   **地址划分**：
    *   **标记 (Tag)**：除块内偏移外的所有高位。由于没有索引字段，访问时需与所有行的标记并行比较 [13, 14]。
    *   **块内偏移 (Offset)**：低位，块内寻址 [9, 13]。

### 具体计算数值例

**已知条件**：
*   主存地址：32 位 [15]。
*   Cache 数据区容量：64 KB [15]。
*   块 (Block) 大小：16 字节 [15]。
*   映射方式：**直接映射** [5]。

**手算过程**：
1.  **块内偏移位 (Offset)**：块大小为 $16\text{B} = 2^4\text{B}$，故需要 **4 位** [2, 5]。
2.  **Cache 总行数**：总容量 / 块大小 = $64\text{KB} / 16\text{B} = 4\text{K}$ 行 ($2^{12}$ 行) [5]。
3.  **行索引位 (Index)**：行数为 $2^{12}$，故需要 **12 位** [2, 5]。
4.  **标记位 (Tag)**：总地址位数 - 索引位 - 偏移位 = $32 - 12 - 4 =$ **16 位** [2, 5]。

**地址逻辑划分图**：
| 标记 (Tag) | 行索引 (Index) | 块内偏移 (Offset) |
| :--- | :--- | :--- |
| 16位 (31-16) | 12位 (15-4) | 4位 (3-0) |

**若改为 4 路组相联映射**：
*   **组数**：总行数 / 每组行数 = $4\text{K} / 4 = 1\text{K}$ 组 ($2^{10}$ 组) [9, 16]。
*   **组索引位**：需要 **10 位** [16, 17]。
*   **标记位**：$32 - 10 - 4 = 18$ 位 [16]。

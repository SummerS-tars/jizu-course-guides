### 写直达 vs 写回
*   **写直达 (Write-through)**：在写命中时，CPU 同时将数据写入 Cache 和主存 [1-3]。这种方式实现简单，能始终保持主存与 Cache 的一致性，但由于主存速度慢，每次写操作都会增加 CPI 负担 [1, 2]。
*   **写回 (Write-back)**：写命中时仅更新 Cache，并为该行设置“脏位 (Dirty bit)”，只有当该块被替换出 Cache 时才将其写回主存 [2-4]。这种策略能显著减少对主存的访问频率，节省带宽，是现代高性能处理器的常用方案 [2, 4, 5]。

### 多核处理器基本架构
*   **核心理念**：随着单核性能提升受阻，多核架构通过在单芯片上集成多个处理器核（Core）来提升并行计算能力 [6, 7]。
*   **存储模型**：
    *   **集中式共享存储 (UMA)**：所有核通过总线或交换网络共享统一的物理内存，访问延迟基本一致 [8, 9]。
    *   **分布式共享存储 (NUMA)**：每个核拥有本地内存，通过互连网络统一编址，访问本地内存快而访问远程内存慢 [8, 10]。
*   **互连结构**：多核之间通过总线、交叉开关或片上网络 (NoC) 进行通信，以实现数据交换和缓存一致性维护 [11, 12]。

### Cache 在存储层次中的位置
*   **物理位置**：Cache 位于 CPU 寄存器与主存 (DRAM) 之间，通常直接集成在 CPU 芯片内部以获得极高的访问速度 [13-15]。
*   **层级设计**：现代处理器采用多级 Cache 结构，其中 L1（分为指令和数据 Cache）和 L2 通常由核心私有，而更大容量的 L3 Cache 则由所有核心共享 [2, 16, 17]。
*   **功能定位**：它作为主存内容的副本，利用程序的局部性原理缓存频繁访问的数据，旨在缓解 CPU 与主存之间的“内存墙”性能缺口 [13, 14, 18]。

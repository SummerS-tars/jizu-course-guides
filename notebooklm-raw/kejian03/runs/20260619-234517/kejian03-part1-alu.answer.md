根据**课件 03**与 **Lab1 实验报告**，关于 ALU 架构设计及其与实验的对照讲解如下：

### 1. ALU 结构：从 1-bit 到 n 位
*   **1-bit ALU**：是 ALU 的基本构成单位，内部包含一个**全加器（FA）**以及用于选择不同操作（如“与”、“或”、“加”）的**多路选择器（MUX）** [1, 2]。
*   **串行 n 位 ALU**：通过将 1-bit ALU 首尾相接形成，低位的进位输出（CarryOut）连接到高位的进位输入（CarryIn），这种结构简单但进位传递速度慢 [1, 3]。
*   **并行 n 位 ALU**：采用**先行进位（CLA）**技术，通过“进位生成”和“进位传递”函数并行产生各级进位，大幅降低了运算延迟 [4-6]。

### 2. ALUop 控制与功能选择
*   **功能选择**：ALU 通过操作控制端（**ALUop**）来决定执行的具体功能 [7, 8]。例如，3 位的 ALUop 最多可支持 8 种操作（如 `000` 为 And，`010` 为 Add，`110` 为 Subtract） [7, 9]。
*   **标志位生成**：在计算结果的同时，ALU 会生成基本条件码，如**零标志（ZF）**、**溢出标志（OF）**、进位标志（CF）和符号标志（SF） [8, 9]。
*   **核心逻辑**：所有复杂的定点运算（乘、除等）本质上都是在**加法器和移位器**的基础上，配合控制逻辑实现的 [7, 10]。

### 3. 与 Lab1 EX 阶段 ALU 的对照点
*   **逻辑实现形式**：课件 03 侧重于底层的门电路与先行进位逻辑设计 [4, 8]；而在 Lab1 中，ALU 被实现为 EX 阶段的一个**组合逻辑模块**，通常直接使用 SystemVerilog 的算术运算符实现，由综合工具自动生成逻辑 [11, 12]。
*   **控制信号映射**：Lab1 译码阶段产生的 `alu_op` 控制信号与课件中的 `ALUop` 概念完全对应，用于在执行阶段驱动 `core_alu.sv` 模块进行运算选择 [12, 13]。
*   **指令扩展**：Lab1 的 ALU 不仅要支持基础的加减（add/sub）和逻辑运算（and/or/xor），还需支持 RISC-V 特有的 **W 后缀指令**（如 `addw`, `subw`），这些指令在 64 位流水线中需要对结果进行 32 位符号扩展 [14, 15]。
*   **处理能力**：课件示例多以 32 位为例 [16, 17]，而 Lab1 要求 CPU 支持 **64 位**算术运算 [14]。

### 1. 缓存一致性 vs. 存储连贯性

*   **缓存一致性 (Cache Coherence)**：解决的是**同一存储单元**在多个私有缓存中的副本同步问题，确保所有核看到的该地址数据是一致的最新值 [1-3]。
*   **存储连贯性 (Memory Consistency)**：解决的是**不同存储单元**访问操作的**定序问题**，定义了写操作在多核间的可见性顺序（即全局访存顺序） [4-6]。

### 2. MESI 协议四状态含义

| 状态 | 名称 | 含义描述 |
| :--- | :--- | :--- |
| **M** | Modified (已修改) | 当前核独有且数据已修改，与主存不一致 [6-8]。 |
| **E** | Exclusive (独占) | 当前核独有且数据干净（与主存一致） [6-8]。 |
| **S** | Shared (共享) | 多个核共享该块，且数据与主存一致 [6, 8, 9]。 |
| **I** | Invalid (无效) | 该缓存行数据无效 [6, 8, 9]。 |

### 3. 自旋锁场景下的 Fence 指令

在宽松一致性模型下，硬件为优化性能可能允许访存指令**乱序执行** [5, 6]。
*   **加锁后加 Fence**：防止临界区内的代码被重排到获取锁（Lock）操作之前，确保只有拿到锁后才访问共享数据 [6, 10]。
*   **解锁前加 Fence**：防止临界区内的写操作被重排到释放锁（Unlock）之后，确保所有数据修改在锁释放前对其他核可见 [6, 10]。

### 4. AMO 与 LR/SC 的原子性直觉

原子性（Atomicity）的本质是保证“读取-修改-写回”序列**不可被中断**且不被其他核介入 [11, 12]。
*   **AMO (内存原子操作)**：在硬件层面将整个过程封装为**单条指令**，在总线上表现为一次不可分割的操作 [13-15]。
*   **LR/SC (加载保留/条件存储)**：通过“标记+校验”实现逻辑原子。**LR** 在地址上设保留标记，**SC** 仅在标记未被破坏（即期间无其他核写入）时才写入成功 [13, 16, 17]。

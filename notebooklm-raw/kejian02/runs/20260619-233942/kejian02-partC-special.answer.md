根据课件 02 及 Week5 期中复习资料，关于 IEEE 754 单精度浮点数（SP）的特殊值与异常处理汇总如下：

### 1. 特殊值编码规则对比表

| 特殊值类型 | 阶码 (E) | 尾数 (f) | 含义与真值公式 | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **±0** | 全 0 | 全 0 | 符号位决定正负，数值为 0 | [1, 2] |
| **非规格化数** | 全 0 | 非全 0 | $±(0.f) \times 2^{-126}$（隐藏位为 0） | [2-4] |
| **±∞** | 全 1 (255) | 全 0 | 表示数值超出表数范围（上溢） | [1, 2] |
| **NaN (非数)** | 全 1 (255) | 非全 0 | 表示未定义或非法的运算结果 | [2, 5] |

*   **SNaN vs QNaN**：**SNaN** (Signaling NaN) 为“发信号非数”，最高有效位通常为 0，会触发浮点异常 [6]。**QNaN** (Quiet NaN) 为“不发信号非数”，最高有效位通常为 1，在运算中静默传播而不产生异常 [5, 6]。
*   **非规格化数**：用于处理“阶码下溢”，填补了最小规格化数与 0 之间的空隙，使精度平滑下降 [2, 4]。

### 2. 特殊值运算规则

*   **NaN 的传播**：任何涉及 NaN 的操作，其结果通常仍为 NaN（如 `NaN + x = NaN`） [5]。
*   **无穷大 (∞) 运算**：
    *   **∞ + 有限数 = ∞** [5]。
    *   **5.0 / 0 = +∞**（注意：浮点数除以 0 得到无穷大，不触发溢出异常；而整数除以 0 是异常） [5, 7]。
*   **非法运算（产生 NaN）**：
    *   **∞ - ∞ = NaN** 或 **∞ / ∞ = NaN** [5, 6]。
    *   **0 / 0 = NaN** 或 **负数开平方根** [5, 6]。

### 3. 数值例分析（单精度 SP）

*   **运算例 A：判断 `0.5 / 0.0` 的结果**
    *   **分析**：0.5 是规格化数，0.0 是特殊值（阶码和尾数全 0） [1, 8]。
    *   **结果**：依据 IEEE 754 规则，除数为 0 且被除数非 0，结果为 **+∞** [5]。
*   **运算例 B：判断 `(1.5e38 + 1.5e38)` 的结果**
    *   **分析**：单精度最大值约为 $3.4 \times 10^{38}$ [9]。
    *   **结果**：两数相加超出最大阶码范围，发生**阶码上溢**，结果符号位为 0，阶码全 1，尾数全 0，即 **+∞** [7, 10]。

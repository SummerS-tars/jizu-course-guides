### 1. 浮点规格化与位域结构
*   **基本位域**：浮点数由符号位 (S)、阶码 (E) 和尾数 (M) 组成 [1, 2]。
    *   **单精度 (SP, 32位)**：S(1) + E(8) + M(23) [1, 2]。
    *   **双精度 (DP, 64位)**：S(1) + E(11) + M(52) [1, 2]。
*   **阶码偏置 (Bias)**：为了方便比较大小，阶码采用移码表示 [3, 4]。
    *   **SP Bias = 127**；**DP Bias = 1023** [1, 5]。
*   **规格化形式**：二进制浮点数统一表示为 $(-1)^S \times 1.f \times 2^{E-Bias}$ [5, 6]。
    *   **隐藏位**：规格化数的整数部分总是“1”，因此不显式存储，从而为尾数多争取 1 位精度（SP 实际精度为 24 位）[1, 6]。

### 2. 完整数值例：互转手算过程

#### A. 十进制 $\rightarrow$ IEEE 754 单精度：以 $-12.75$ 为例 [7]
1.  **确定符号**：负数，故 **$S = 1$**。
2.  **进制转换**：
    *   整数部分：$12 = 8 + 4 = 1100_2$。
    *   小数部分：$0.75 = 0.5 + 0.25 = 0.11_2$。
    *   合并得：$1100.11_2$。
3.  **规格化**：向左移动小数点 3 位，得 $1.10011_2 \times 2^3$。
4.  **计算阶码 (E)**：指数为 3，则 $E = 3 + 127 = 130$。
    *   转换为二进制：$130 = 128 + 2 = 1000\ 0010_2$。
5.  **拼接位域**（尾数 $f$ 舍弃整数位“1”并补齐至 23 位）：
    *   `S=1 | E=10000010 | f=10011000000000000000000`
    *   **最终结果**：`C14C0000H`。

#### B. IEEE 754 单精度 $\rightarrow$ 十进制：以 `BEE00000H` 为例 [8]
1.  **展开二进制**：`1 01111101 11000000000000000000000`。
2.  **解析位域**：
    *   **$S = 1$**：负数。
    *   **$E = 0111\ 1101_2 = 125_{10}$**。
    *   **$f = 110...0_2$**。
3.  **计算指数与真值**：
    *   指数偏移：$125 - 127 = -2$。
    *   还原隐藏位：尾数为 $1.11_2 = 1 + 2^{-1} + 2^{-2} = 1.75$。
4.  **最终数值**：$-1.75 \times 2^{-2} = -0.4375$。

### 3. 精度分布不均匀性直觉
*   **直觉模型**：浮点数在数轴上的分布是**近零密集、远零稀疏**的 [4, 9]。
*   **原因分析**：
    *   阶码 $E$ 决定了数值所在的“段”（数量级），尾数 $M$ 决定了该段内的“刻度” [10]。
    *   由于尾数位数固定（如 SP 只有 23 位），在每一段内能表示的刻度总数是固定的 [4]。
    *   随着阶码增大，每一刻度代表的绝对物理间隔会随之呈指数级倍增（例如从 $2^{-126}$ 跨度跳到 $2^{127}$），导致大数之间的缝隙越来越大 [9, 11]。

根据课件 02 和 Week5 的核心内容，IEEE 754 浮点体系的要点讲解如下：

### 1. 体系结构与规格化
*   **基本公式**：真值 = $(-1)^S \times (1.f) \times 2^{E - \text{Bias}}$，其中 $1.f$ 为规格化尾数 [1, 2]。
*   **位域分配**：
    *   **单精度 (SP, 32位)**：1位符号 (S) + 8位阶码 (E) + 23位尾数 (M) [1, 3]。
    *   **双精度 (DP, 64位)**：1位符号 (S) + 11位阶码 (E) + 52位尾数 (M) [1, 3]。
*   **阶码偏置 (Bias)**：单精度偏置值为 127，双精度为 1023 [1, 4]。使用偏置常数是为了将指数转换为正整数，便于硬件直接按位比较大小 [5, 6]。
*   **隐藏位技术**：规格化尾数的小数点前总是 1，故该“1”不显式存储，从而为尾数多争取了 1 位精度（单精度实际精度为 24 位）[2, 7, 8]。

### 2. 完整数值例：十进制 $\leftrightarrow$ 单精度 (SP) 互转
#### 例 A：将十进制数 **-12.75** 转换为 SP 机器数 [9]
1.  **确定符号**：负数，故 $S = 1$ [9]。
2.  **二进制转换**：$12 = 1100_2$，$0.75 = 0.11_2$，合起来为 $1100.11_2$ [9, 10]。
3.  **规格化**：向左移动小数点，$1100.11 = 1.10011 \times 2^3$，指数为 3 [9]。
4.  **计算阶码**：$E = 3 + 127 = 130 = 1000 0010_2$ [9, 11]。
5.  **合并字段**：$S(1) + E(10000010) + M(1001100...0)$ [9]。
6.  **最终结果**：二进制为 `1100 0001 0100 1100 0000...0`，即十六进制 **C14C 0000H** [9]。

#### 例 B：将 SP 机器数 **BEE0 0000H** 还原为十进制 [12]
1.  **展开二进制**：`1 01111101 110 0000...0` [12]。
2.  **符号**：$S = 1$，为负数 [12]。
3.  **阶码**：$E = 01111101_2 = 125$。真实指数 $= 125 - 127 = -2$ [12]。
4.  **尾数**：$f = 110...0$，还原隐藏位后为 $1.11_2 = 1 + 0.5 + 0.25 = 1.75$ [12]。
5.  **真值**：$-1.75 \times 2^{-2} = -0.4375$ [12]。

### 3. 精度分布不均匀性直觉
*   **近零密集，远零稀疏**：浮点数在数轴上的分布是不均匀的 [7, 13]。
*   **直觉理解**：尾数部分提供的位数固定（单精度 23 位），这意味着在同一个阶码范围内，可表示的数值个数是恒定的 [4]。
*   **绝对误差随阶码增大**：当指数 $E$ 增加时，相邻两个浮点数之间的间隔（即单位最低有效位 ULP 的权值）呈指数级增长 [7, 14]。
*   **结果**：在接近 0 的区域，数值分布极度密集；随着数值绝对值增大，两个可表示数之间的“空隙”越来越大，导致大数运算的精度损失更严重 [4, 13]。

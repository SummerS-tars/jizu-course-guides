根据**课件 02**及 **Week5 记录**，IEEE 754 单精度浮点数的特殊值与异常处理规则总结如下：

### 1. 特殊值编码规则对比表
在单精度（32位）格式中，阶码（E）为 8 位，偏置值为 127 [1, 2]。

| 值类型 | 阶码 (E) | 尾数 (M) | 真值 / 含义 | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **正/负零** | 全 0 | 全 0 | $\pm 0$（符号位决定） | [课件02-19, 227] |
| **非规格化数** | 全 0 | 非全 0 | $(-1)^S \times 0.M \times 2^{-126}$（用于处理阶码下溢） | [课件02-21, 227] |
| **规格化数** | 1 ~ 254 | 任意 | $(-1)^S \times 1.M \times 2^{E-127}$ | [课件02-26, 227] |
| **正/负无穷** | 全 1 (255) | 全 0 | $\pm \infty$（如除以 0 的结果） | [课件02-19, 227] |
| **NaN (非数)** | 全 1 (255) | 非全 0 | 表示非法运算结果（如 $0/0$） | [课件02-21, 227] |

**NaN 的分类：**
*   **QNaN (Quiet NaN)**：尾数最高位为 1。产生时不触发异常，在表达式中静默传播 [3]。
*   **SNaN (Signaling NaN)**：尾数最高位为 0。产生或参与运算时会触发硬件异常陷阱 [3]。

### 2. 特殊值运算规则
浮点运算不仅产生特殊值，也能以特殊值作为操作数 [4, 5]：

*   **$\infty$ 相关**：
    *   $\infty + \text{有限数} = \infty$ [4]。
    *   $\infty + \infty = \infty$ ；但 $\infty - \infty = NaN$ [4, 6]。
    *   $\text{有限数} / 0 = \pm \infty$ （注意：整数除 0 是溢出异常，浮点除 0 结果是 $\infty$） [4]。
*   **NaN 相关**：
    *   **NaN 传播**：任何涉及 NaN 的算术运算，结果均为 NaN [4]。
    *   **无效运算**：$0/0$、$\infty / \infty$、$\infty \times 0$ 或对负数求平方根均产生 NaN [4, 6]。

### 3. 数值判断示例

**示例 A：**
*   **操作**：`0x7F800000` (操作数1) + `0x3F800000` (操作数2)
*   **判断**：
    *   `0x7F800000` 的阶码为 255，尾数为 0，表示 $+\infty$ [7]。
    *   `0x3F800000` 的阶码为 127 ($E-127=0$)，尾数为 0，表示 $1.0$。
*   **结果**：$+\infty + 1.0 = +\infty$。结果机器码仍为 `0x7F800000` [4]。

**示例 B：**
*   **操作**：`0xFFC00000` (操作数1) $\times$ `0x00000000` (操作数2)
*   **判断**：
    *   `0xFFC00000` 的阶码为 255，尾数不为 0（且最高位为 1），表示 $QNaN$ [3, 8]。
    *   `0x00000000` 为 $+0$ [7]。
*   **结果**：根据 NaN 传播规则，任何数与 NaN 运算结果均为 $NaN$ [4]。

这些规则确保了浮点运算在极端情况下（如溢出或非法输入）仍能继续执行而不至于直接崩溃 [1, 5]。对于下一步，你想了解浮点数的舍入模式还是具体的加减对阶步骤？

根据课件 02 及相关周笔记，这里为你整理了 5 组最易混淆的核心概念对比：

### 1. 整数编码：原码 vs 补码 vs 移码
| 特性 | 原码 (Sign-Magnitude) | 补码 (Two's Complement) | 移码 (Excess/Biased) |
| :--- | :--- | :--- | :--- |
| **符号位** | 0为正，1为负 [1] | 0为正，1为负 [2] | 与补码符号位相反 [3] |
| **0 的表示** | 不唯一（+0 和 -0）[1, 4] | **唯一**（全 0）[2, 5] | 唯一 [6] |
| **主要用途** | 浮点数的尾数 [1, 5] | **定点整数运算** [5, 7] | **浮点数阶码**（便于比较） [6, 8] |

**来源：** (课件02 / Week4-5 / Week16 复习) [1, 2, 5, 7]

---

### 2. 浮点数状态：规格化 vs 非规格化数 (SP)
| 特性 | 规格化数 (Norms) | 非规格化数 (Denorms) |
| :--- | :--- | :--- |
| **阶码字段** | 1 ~ 254 [7, 9] | **全 0** [7, 9] |
| **隐含位** | 1 [7, 10] | **0** [7, 11] |
| **作用** | 表示绝大多数实数 [9] | 填补 0 与最小规格化数间的间隙，平滑下溢 [11, 12] |

**来源：** (课件02 / Week4-5 / Week16 复习) [7, 9]

---

### 3. 特殊非数：SNaN vs QNaN
| 特性 | 发信号非数 (SNaN) | 不发信号非数 (QNaN) |
| :--- | :--- | :--- |
| **尾数最高位** | 通常为 0 [13, 14] | 通常为 1 [13, 14] |
| **触发异常** | **会**触发浮点异常 [14, 15] | **不会**触发，静默传递结果 [14, 15] |
| **作用** | 捕获未初始化的变量或非法操作 [15] | 允许计算在异常后继续执行 [15] |

**来源：** (课件02 / Week5) [14, 15]

---

### 4. 字节序：大端 (Big Endian) vs 小端 (Little Endian)
| 方式 | 存放规则 | 典型架构 / 协议 |
| :--- | :--- | :--- |
| **大端** | **最高有效字节 (MSB)** 存放在低地址 [16, 17] | 网络协议、MIPS (可调) [16, 17] |
| **小端** | **最低有效字节 (LSB)** 存放在低地址 [16, 17] | **x86、RISC-V** [17, 18] |

**来源：** (课件02 / Week4-5) [16-18]

---

### 5. IEEE 754 精度：单精度 vs 双精度位域
| 格式 | 总位数 | 阶码 (Exponent) | 尾数 (Significand) | 偏置值 (Bias) |
| :--- | :--- | :--- | :--- | :--- |
| **单精度 (float)** | 32 [7, 10] | 8 位 [7, 10] | 23 位 (实 24) [7, 10] | 127 [7, 10] |
| **双精度 (double)** | 64 [7, 10] | 11 位 [7, 10] | 52 位 (实 53) [7, 10] | 1023 [7, 10] |

**来源：** (课件02 / Week4-5 / Week16 复习) [7, 10]

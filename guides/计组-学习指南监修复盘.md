## 3. 可推广样板：Week1-3 §2.2

Week1-3 §2.2 的优势不只是“写得详细”，而是内部结构完整：

1. **先定义读者问题**：单周期 CPU 一周期内到底要走完什么，而不是先背模块名。
2. **状态/组合分层**：PC、RF、DM 写口等状态部件与 IM、ALU、CU、MUX 等组合路径分清楚。
3. **控制信号按用途记**：把 RegWrite/MemWrite 归为写使能，把 ALUSrc/MemtoReg 归为 MUX 选择，比孤立背 0/1 更容易迁移。
4. **关键路径讲“为什么”**：`lw` 不是因为常见而关键，而是经过 IM→RF→ALU→DM→RF 的最长链。
5. **图前图后都有解释**：先说明怎么带着 `lw` 读图，图后再提醒并非所有箭头每条指令都走。
6. **最后落到真值表、模板、易错点**：读者能直接用于开卷解题。

可推广模板：

```text
本节要回答
→ 先看问题/动机
→ 术语与部件边界
→ 机制步骤或状态迁移
→ 图/表
→ 图后读法
→ 解题模板或数值例
→ 易错点
→ 小结与下一节承接
```

优先推广到：Week10-11 §2.2/§2.3、Week13-14 §2.4、课件 06 Part C/D、课件 05b Part 3/4。

# 作业一

## 题目1

甲、乙为以不同方式实现同一个指令集（一共包含有A、B、C、D四种类型的指令）的机器，时钟频率分别为1GHz和1.5GHz。甲、乙都提供了各自的编译器C1和C2，它们对于同一个测试程序编译出来的指令总数相同，占比不同。

| 指令类型 | 甲的CPI | 乙的CPI | C1编译 | C2编译 |
| -------- | ------- | ------- | ------ | ------ |
| A        | 1       | 1       | 40%    | 10%    |
| B        | 2       | 2       | 30%    | 60%    |
| C        | 3       | 6       | 20%    | 20%    |
| D        | 4       | 7       | 10%    | 10%    |

请分析下列问题：

- 采用C1编译器，两者运行速度哪个快？
- 采用C2编译器，两者运行速度哪个快？

## 题目1解答

- 采用C1编译器，两者运行速度哪个快？

甲的 $\text{CPI}=40\%\times 1+30\%\times 2+20\%\times 3+10\%\times 4=2$

乙的 $\text{CPI}=40\%\times 1+30\%\times 2+20\%\times 6+10\%\times 7=2.9$

设指令数为 $I$：

甲的执行时间 $\dfrac{2I}{1\text{GHz}}$

乙的执行时间 $\dfrac{2.9I}{1.5\text{GHz}}$

$\dfrac{2I}{1\text{GHz}}=\dfrac{3I}{1.5\text{GHz}}>\dfrac{2.9I}{1.5\text{GHz}}$

故如果采用 C1 编译器，乙的运行速度更快。

- 采用C2编译器，两者运行速度哪个快？

甲的 $\text{CPI}=10\%\times 1+60\%\times 2+20\%\times 3+10\%\times 4=2.3$

乙的 $\text{CPI}=10\%\times 1+60\%\times 2+20\%\times 6+10\%\times 7=3.2$

$\dfrac{2.3I}{1\text{GHz}}=\dfrac{3.45I}{1.5\text{GHz}}>\dfrac{3.2I}{1.5\text{GHz}}$

故如果采用 C2 编译器，乙的运行速度更快。

## 题目2

假定机器M的时钟频率为1.5GHz，某程序P在机器M上的执行时间为12秒钟。对P优化时，将其所有的乘4指令都换成了一条左移2位的指令，得到优化后的程序P'。已知在M上乘法指令的CPI为5，左移指令的CPI为2，P的执行时间时P'执行时间的1.2倍，则P中有多少条乘法指令被替换成了左移指令被执行？

## 题目2解答

原有的执行时间 $t=12\text s$，改进后的执行时间 $t'=\dfrac{t}{1.2}=10\text s$。

时间差 $\Delta t=t-t'=2\text s$

周期差 $\Delta \text C=1.5\text{GHz}\times \Delta t=3\times 10^9$

指令数 $I=\dfrac{\Delta \text C}{\Delta \text{CPI}}=\dfrac{3\times 10^9}{5-2}=10^9$

故 P 中有十亿条乘法指令被替换成了左移指令执行。

## 题目3

优化处理器时加入一个浮点单元

1. 该浮点单元使浮点运算速度提高到3倍，原来浮点运算占用时间为源程序执行时间的30%，不考虑对其它指令的影响，最终的加速比是多少？
2. 假设上述浮点单元会降低访存访问速度，减缓倍数为2。原来数据缓存访问时间占源程序执行时间的20%，现在总加速比是多少？
3. 实现新的浮点单元后，浮点运算上花费的执行时间占多大比例？数据访存占多大比例？

## 题目3解答

- 该浮点单元使浮点运算速度提高到3倍，原来浮点运算占用时间为源程序执行时间的30%，不考虑对其它指令的影响，最终的加速比是多少？

设原本程序执行时间为 $t$。

加速后程序执行时间 $t'=(1-30\%)t+\dfrac{30\%}{3}t=0.8t$

故加速比 $\dfrac{t}{t'}=1.25$。

- 假设上述浮点单元会降低访存访问速度，减缓倍数为2。原来数据缓存访问时间占源程序执行时间的20%，现在总加速比是多少？

设原本程序执行时间为 $t$。

变更后后程序执行时间为 $t'=(1-30\%-20\%)t+2\times 20\% t+\dfrac{30\%}{3}t=t$

故加速比为 1。

- 实现新的浮点单元后，浮点运算上花费的执行时间占多大比例？数据访存占多大比例？

浮点运算占比 $\dfrac{\frac{30\%}{3}t}{t'}=10\%$

数据访存占比 $\dfrac{2\times 20\%t}{t'}=40\%$

## 题目4

一台专用的数字信号处理计算机，运行程序P，乘法运算操作占程序运行时间的30%。除法指令占15%。现在对该系统进行优化，有两种方案：a) 采用除法硬件加速，可以使除法运算性能提高到10倍；b) 改进所有乘法和除法指令，使运算速度提高到5倍。

- 比较两种方案，哪种加速效果更好；
- 如果要使整个程序的运算性能提高到原来的1.4倍，两种优化方案中，各自需要将对应指令性能提升多少？

## 题目4解答

- 较两种方案，哪种加速效果更好；

设原本运行时间为 $t$，则：

a 方案运行时间 $t_a=(1-15\%)t+\dfrac{15\% t}{10}=86.5\% t$

b 方案运行时间 $t_b=(1-30\%-15\%)t+\dfrac{35\%+15\%}{5}t=64\%t$

由于 $t_b<t_a$，故 b 方案加速效果更好。

- 如果要使整个程序的运算性能提高到原来的1.4倍，两种优化方案中，各自需要将对应指令性能提升多少？

注意到 $1/(1-15\%)<1.4$，故 a 方案无法实现题目要求的优化。

考虑 b 方案，不妨设 b 方案将对应的乘除指令性能同时提高到原来的 $x$ 倍，则：

$t_b=\dfrac{t}{1.4}=(1-30\%-15\%)t+\dfrac{35\%+15\%}{x}t$

得到方程 $\dfrac{1}{1.4}=0.55+\dfrac{0.45}x$

解得 $x=\dfrac{63}{23}$

故 a 方案无法实现题目要求的优化，

而 b 方案需要将对应指令性能提高到原来的 $\dfrac{63}{23}$ 倍。

# 作业二

## 题目1

在以下循环中，找出所有 **输出相关**和**反相关**。通过 **重命名** 来消除 **输出相关** 和 **反相关** ，并写出重命名后的真相关。

```c
for (i = 0; i < 100; i++) {
    A[i] = A[i] * B[i]; 	/* S1 */
    B[i] = B[i] * C[i]; 	/* S2 */
    A[i] += D[i]; 		    /* S3 */
    C[i] = D[i] * A[i]; 	/* S4 */
}
```

## 题目1解答

由于在循环中，所有数组访问的下标都是 [i]，故不存在循环携带相关，只需分析单一循环语句内部的数据相关。

观察四个语句读取与写入的变量：

- S1 读 A、B，写 A。
- S2 读 B、C，写 B。
- S3 读 A、D，写 A。
- S4 读 A、D，写 C。

反相关（WAR）：

- S1 读 B 之后，S2 要写 B，即 S1 和 S2 关于 B[i] 存在反相关。
- S2 读 C 之后，S4 要写 C，即 S2 和 S4 关于 C[i] 存在反相关。

输出相关（WAW）：

- S1 写 A 之后，S3 要写 A，即 S1 和 S3 关于 A[i] 存在输出相关。

采用寄存器重命名技术可以消除反相关和输出相关，代码如下：

```c
for (i = 0; i < 100; i++) {
    A1[i] = A[i] * B[i]; 	/* S1 */
    B1[i] = B[i] * C[i]; 	/* S2 */
    A2[i] = A1[i] + D[i]; 	/* S3 */
    C[i] = D[i] * A2[i]; 	/* S4 */
}
```

观察四个语句读取和写入的变量：

- S1 读 A、B，写 A1。
- S2 读 B、C，写 B1。
- S3 读 A1、D，写 A2。
- S4 读 A2、D，写 C。

可以看出，真相关（RAW）：

- S1 写 A1 之后，S3 要读 A1，故 S1 和 S3 关于 A1[i] 存在真相关。
- S3 写 A2 之后，S4 要读 A2，故 S3 和 S4 关于 A2[i] 存在真相关。

## 题目2

在经典的5段顺序流水线中，load-use数据冒险是一类典型的数据冒险，回答下列问题：

（1）如果使用编译器来解决load-use数据冒险，该如何处理？（2分）

（2）如果使用硬件检测，则检测条件是什么？（2分）

（3）对下面的指令代码进行优化，以减少Load-use数据冒险。（4分）

```assembly
lw  $2, [b]
lw  $3, [c]
add $1, $2, $3
sw  $1, [a]
lw  $5, [e]
lw  $6, [f]
xor $4, $5, $5
sw  $4, [d]
```

## 题目2解答

- 如果使用编译器来解决load-use数据冒险，该如何处理？

编译器主要有两种方法解决 load-use 数据冒险：

1. 指令调度（重排序），编译器通过将 load 指令提前，或者在 load 指令和紧接着使用该数据的指令之间，插入其它不相关的指令，以拉开 load 指令和 use 指令的距离，等到 use 指令执行时，数据已经从内存读出并写回寄存器了。

2. 如果编译器在调度时找不到可以插入的无关指令，编译器就会直接在 load 指令和后续依赖指令之间插入一条或多条空指令，强行延迟后续指令的执行。

- 如果使用硬件检测，则检测条件是什么？

在 ID 到 EX 阶段流水线寄存器是一条 load 指令，且 ID 到 EX 阶段流水线寄存器 load 指令的目标寄存器和 IF 到 ID 阶段流水线寄存器指令的其中一个源寄存器编号相同。

- 对下面的指令代码进行优化，以减少Load-use数据冒险。

```assembly
lw  $2, [b]
lw  $3, [c]
add $1, $2, $3
sw  $1, [a]
lw  $5, [e]
lw  $6, [f]
xor $4, $5, $5
sw  $4, [d]
```

唯一一处 load-use 冒险发生在第二行和第三行，一种可行的重排方式如下：

```assembly
lw  $2, [b]
lw  $3, [c]
lw  $5, [e]
lw  $6, [f]
add $1, $2, $3
xor $4, $5, $5
sw  $1, [a]
sw  $4, [d]
```

# 作业三

## 题目1

给定以下的假设，试分别比较直接映象 Cache 和两路组相联 Cache 的平均访问时间以及 CPU 时间。(10 分)
**假设条件：**

- 理想 Cache（命中率为 100%）情况下的 CPI 为 1.0，平均每条指令访存 1.2 次
- 两者 Cache 容量均为 64KB，块大小都是 32 字节
- 直接映象 Cache 的时钟周期为 1ns，组相联 Cache 中的多路选择器使 CPU 的时钟周期比直接映象 Cache 增加了 10％
- 这两种 Cache 的缺失代价都是 100ns
- 两种 Cache 的命中时间都为 1 个时钟周期
- 64KB 直接映象 Cache 的缺失率为 3％，64KB 两路组相联 Cache 的缺失率为 1％

## 题目1解答

平均访问时间：

直接映像 Cache： $t_1=1\text{ns}+100\text{ns}\times 3\%=4\text{ns}$

两路组相联 Cache： $t_2=1.1\text{ns}+100\text{ns}\times 1\%=2.1\text{ns}$

CPU 时间：

直接映像 Cache：$T_1=1\text{ns}\times I+100\text{ns}\times 1.2\times 3\%\times I=4.6\text{ns}\times I$

两路组相联 Cache：$T_2=1.1\text {ns}\times I+100\text{ns}\times 1.2\times 1\%\times I=2.3\text{ns}\times I$

由此可见，两路组相联虽然导致时钟周期略慢，但由于使得缺失率降低，最终整体性能获得提升。

## 题目2

按字节编址的 24 位地址的机器上有一个存放 32KB 数据的 cache（块大小为 32B），和拥有 16 个表项的 TLB
**问题：**

## 题目2解答

- 若该 cache 为全相联，则 24 位地址的各个字段的作用是什么？若是直接映射呢？

cache 的行数为 $\dfrac{32\text{KB}}{32\text B}=1024=2^{10}$

cache 块的大小为 $2^5 \text B$

因此，若是全相联，24 位地址的高 19 位是标记，作用是在整个 cache 块中进行关联比较，以判断对应的主存块是否命中；而低 5 位是块内偏移，用于在命中的 32 字节块中寻址对应的字节。

若是直接映射，24 位地址的高 9 位是标记，作用是比较判断对应的主存块是否命中；中间 10 位是行号，用于直接定位到 cache 中唯一的硬件行；低 5 位是块内地址，用于在命中的块中寻找字节。

- 若 cache 为 2 路组相联和 FIFO 替换算法，起始为空，进程 P 顺序地修改了地址 1024 到 0 的数据，每次修改一个字节，总共请求次数是多少，其中缺失了多少次？

注意到 $1024-0+1=1025$ ，所以总共的请求次数是 1025 次。

修改地址 1024 时缺失一次。

后续每请求一次，都会将后续 32 个字节放入 cache 中，因此后续缺失次数是 $1024/32=32$。

因此会在 1025 次请求中发生 33 次冷缺失。

- 若机器的虚拟地址为 32 位，页大小为 16KB，则虚拟地址中哪几位表示虚拟页号，哪几位表示页内地址；若 TLB 为 8 路组相联，则页表项的标志位，组号和物理页框号分别用到几位

由于页大小 $16\text{KB}=2^{14}\text{B}$，故虚拟地址中 $32-14=18$ 位表示虚拟页号，$14$ 位表示页内地址。

由于 $2^4$ 项 TLB 有 $2^3$ 路组相联，组数只有 2 组，故组号只需1位，标志位$18-1=17$位。

物理页框号 $24-14=10$ 位。

- 若 TLB 为 2 路组相联和 LRU 替换算法，起始为空，依次访问虚拟页号 11、19、1、14、11、6、27、22、3、27、4、1，哪些虚拟页号对应的 TLB 会被替换掉，说明理由

由于 $16$ 项 TLB 有 $2$ 路组相联，组数 8 组，依次访问虚拟页号：

1. 访问 11，映射到组 3，缺失，组 3 填入 [11]
2. 访问 19，映射到组 3，缺失，组 3 填入 [19,11]
3. 访问 1，映射到组 1，缺失，组 1 填入 [1]
4. 访问 14，映射到组 6，缺失，组 6 填入 [14]
5. 访问 11，映射到组 3，命中，组 6 填入 [11,19]
6. 访问 6，映射到组 6，缺失，组 6 填入 [6,14]
7. 访问 27，映射到组 3，缺失，组 3 填入 [27,11]，19 被替换
8. 访问 22，映射到组 6，缺失，组 6 填入 [22,6]，14 被替换
9. 访问 3，映射到组 3，缺失，组 3 填入 [3,27]，11 被替换
10. 访问 27，映射到组 3，命中，组 3 填入 [27,3]
11. 访问 4，映射到组 4，缺失，组 4 填入 [4]
12. 访问 1，映射到组 1，命中，组 1 填入 [1]

故被替换掉的虚拟页号是 19、14、11。

## 题目2

**题目描述：**
参考《计算机体系结构量化方法》，在评估互连网络时，**截面带宽**和**聚合带宽**是关键指标。假设我们要为一个拥有64个处理器的系统设计互连网络，每个处理器的链路带宽为 $B$ bytes/sec。请分别计算在以下两种拓扑结构下，该网络的**截面带宽**和**聚合带宽**：
(1) **总线结构**：所有处理器共享一条总线，总线带宽为 $B$。
(2) **二维环网形**：处理器排列为 $8 \times 8$ 的网格，每条链路带宽为 $B$。

并从量化角度讨论，为什么总线结构无法扩展到大规模多处理器系统，而环网形可以？

## 题目2解答

![img](https://ctf.xidian.edu.cn/api/media?hash=2705cd3e8a6780919e62590779ebd6aa57cb419dd2b567e9a4ca3cb8132bb3f5)

总线结构的截面带宽：

将共享一条总线的 64 个处理器对半分，这两半的通信带宽是 $B$，故该网络的截面带宽是 $B$。

总线结构的聚合带宽：

在一条传统的共享总线，同一时刻只能有一个源节点向一个目的节点发送消息，故该网络的聚合带宽是 $B$。

![img](https://ctf.xidian.edu.cn/api/media?hash=b2a1c1f1746da8c49d5df1f40a21f99aff337e905c29feb8ec8b2a37e791290d)

二维环网形的截面带宽：

将 $8\times 8$ 的环网型网格对半分，需要切 16 根线，包括 8 根中间切口和 8 根边缘环回口，故该网络的截面带宽是 $16B$。

二维环网形的聚合带宽：

在二维环网型网格中，每个节点有 4 条出链路，如果每个节点同时和其相邻节点通信，总线可以全部工作，设网络的带宽是无向带宽，则聚合带宽为 $\dfrac{64\times 4}{2}B=128B$。

从量化角度讨论结构的可扩展性，假设处理器数为 $N$。

总线结构的截面带宽为 $B$，聚合带宽为 $B$，是恒定值，在处理器数增加时，每个处理器分到的平均带宽非常少，容易引发严重的排队延迟。

而二维环网形的截面带宽（约）为 $2\sqrt{N}B$，聚合带宽为 $2NB$，在处理器数增加时，虽然通信跳数增加了，但网络的总吞吐能力增长，有利于缓解通信问题。

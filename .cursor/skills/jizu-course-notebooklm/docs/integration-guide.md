## 2. 认知编排原则

### 2.1 先「学什么」，再「怎么学」

每个大模块正文开始前，必须有**全景节**（§1.1）+ **学完你能**（3–5 条可检验能力）：

| 要素 | 要求 |
|------|------|
| 模块解决什么问题 | 一句话 + 与上周衔接 |
| **学完你能** | 3–5 条可检验能力（手算 / 画图 / 分析…） |
| 内部结构预告 | 问题链或 mermaid |
| 自检问题 | §6「读完本章你应能…」与「学完你能」呼应 |

### 2.2 认知阶梯（由浅入深）

```
L0 定位与动机 → L1 全景地图 → L2 符号与概念 → L3 推导与机制
  → L4 数值验证 → L5 工程流程 → L6 串联
```

整合顺序按**读者认知阶梯**，不是 manifest 采集顺序。

### 2.3 三层叙事（每章必查）

**（1）章级叙事线**——§1.3 或各 §2 大节开头一句话说明在整体中的作用

**（2）节级「要回答」**——每 §2.x 首行：`> **本节要回答**：[问题]`

**（3）节间衔接**——每 §2.x 末尾：`> **小结 → 下一节承接**：…`（最后一节可承接 Lab 或下一模块）

### 2.4 重要章节必须拆成子知识点

核心机制章节不能用一个粗粒度小节一笔带过。凡涉及数据通路、流水线、Cache/TLB、异常、一致性、Tomasulo、控制信号等高频考点，§2.x 内部必须继续拆成 `#### 2.x.1`、`#### 2.x.2` 等子知识点，顺序遵循：

1. **先讲动机 / 术语 / 要解决的问题**：首次出现的关键术语、指令或机制，要说明用途、为什么本节要单独讲。
2. **再讲结构与机制**：分清状态部件、组合逻辑、控制信号、输入输出和触发条件。
3. **再给图或表**：mermaid 图、真值表、状态图用于辅助定位，不能替代文字讲解。
4. **图后必须解释怎么读**：说明实线/虚线、条件路径、关键路径、状态迁移或表格列含义，避免读者只看到框图。
5. **最后给解题模板 / 易错点**：将机制落到判题、手算、实验调试或开卷查表步骤。

例如讲单周期 `lw` 关键路径时，应先说明 Load Word 是“从内存读一个 word 到寄存器、常用于把数据搬进 CPU 参与运算”，再解释它为什么比 `add` 多走 DM 并常成为最长路径；不能直接写“`lw` 是关键路径”。

## 3. 计组特定要求

### 3.1 可视化
- 数据通路图、流水线时空图、存储层次图 **必须用 mermaid**
- **≥3 张/模块**（flowchart / graph / stateDiagram / sequenceDiagram）
- 标注关键路径（单周期时钟周期、Page Walk 层级等）
- 图不能替代讲解：图前要有动机与术语，图后要写清读图方法、条件路径和关键控制点。

### 3.2 数值例与难点补充
- 浮点数 IEEE 754（32/64 位）、补码手算
- Cache 地址映射（直接映射/组相联）
- 流水线 CPI、阿姆达尔定律
- **难点**须补：真值表、状态时序、完整手算步骤（勿只给结论）

### 3.3 对比表（≥2 组/模块）
- RISC vs CISC、单周期 vs 多周期 vs 流水线
- 直接映射 vs 组相联 vs 全相联
- 写直达 vs 写回、监听 vs 目录
- 一致性 vs 连贯性、TLB miss vs Page Fault

### 3.4 Lab 专章化

- 周次/课件指南只保留轻量 Lab ref：说明相关 Lab 是哪个、对应本周概念是什么，并指向 `guides/计组-Lab1-6-整合指南.md`。
- Lab 的详细机制、个人实现、代码路径、报告来源、调试坑统一进入 Lab 整合指南，避免周次指南承担 Lab 专章职责。
- Lab 整合指南必须按 Lab 分 Part；每个 Lab 至少包含 **考察知识点**、**我的实现**、**关键代码路径/报告来源**、**易错点/调试方法**、**对应周次/课件**。
- 重要机制需尽量给准确代码路径或短代码片段；若本地代码、报告或 raw 无法核对，明确标注“待核对”，不得编造实现细节。

## 6. 语言与风格

### 6.1 术语与缩写格式

- 术语表及正文**首次出现**时，专业名词采用 **中文（English）**；英文缩写采用 **缩写（English full name，中文）**。
- 高频缩写必须在本指南的术语表或“缩写速查”中有解释入口；正文只在首次出现处补全，避免每次出现都机械展开。
- 表格字段、标题、mermaid/图节点中的缩写也视为**首次出现**：必须在表头、注释、图前说明或同一表格内给出英文全称 / 中文解释；不要只在正文解释而忽略表格和图。
- 计组常见缩写包括但不限于：CPU、ALU、CU、ISA、PC、RF、IM、DM、CPI、IPC、ILP、TLP、SIMD、MIMD、GPU、DMA、MMIO、CSR、SATP、TLB、PTE、VPN、PPN、VA、PA、MMU、AMAT、CDB、ROB、RS、BTB、BHT、RISC、CISC、MIPS、RISC-V、MESI、MSI、NUMA、UMA、NoC、PCIe、RAID。

- **简体中文**，简洁直观，减少与 §1 重复的套话
- 正文主线保留定义、机制、步骤、公式推导；偏解释、类比、读图提示、易错提醒、概念边界说明优先放入引用块，例如 `> **直观理解：** ...`、`> **读图提示：** ...`、`> **易错提醒：** ...`、`> **边界说明：** ...`

## 9. 整理途中持续加入优化规则/建议

课件轨与周次轨都允许在 Review 中吸收更好的监修规则，但必须做成**可复用、可追踪、可回查**的闭环，避免只在单份指南里临时修补。

### 9.1 监修反馈池

每次用户或 Agent 发现可复用问题时，先归入反馈池，再决定回写位置：

| 反馈类型 | 例子 | 回写位置 |
|----------|------|----------|
| 结构缺口 | 缺「本章怎么用」、课件顺序与授课顺序未说明 | 本文 §8、`checklist.md` |
| 计组难点 | Sv39/PTE 权限、MESI 状态迁移、Tomasulo 周期边界 | `guides/计组-课件学习指南监修标准.md` |
| 表达模板 | 追问只有提示、开卷路径不可操作 | 本文 §2、§8 与 checklist |
| 章节个案 | 07b 地址拆位需二轮核算，08 Fence 边界需用户看 | 对应指南「监修自检」与索引 §六 |

### 9.2 局部规则回写

新规则分三层回写：

1. **通用规则**：所有学习指南都适用，写入本文 §2/§6 或 `checklist.md`。
2. **课件轨规则**：只影响 `计组-课件*-学习指南.md`，写入本文 §8 与监修标准。
3. **单章规则**：只影响某一章，写入该指南「监修自检」的二轮复查项，必要时同步 `计组-课件梳理索引.md`。

### 9.3 章节二轮复查

首轮完整后进入二轮复查，不大幅重写已完整内容，优先补**可见缺口**：

- 「本章怎么用」：给出开卷复习路径、先查哪张表、先做哪类手算。
- 三方定位：每章说明课件 PDF、双教材、Lab/周次指南的对应关系。
- 难点可操作：关键处给直观解释、反例、边界条件、数值例、状态机或 mermaid。
- 示例题模板：计算题不得只给答案，必须有题目场景、已知、求、公式、逐步计算、结果解释、易错提醒。
- 术语入口：缩写/专名首次出现需给英文全称、中文解释和本课程用途；MIPS 等辅助案例要说明与 RISC-V/Lab 主线的关系。
- 表格/图节点/标题入口：缩写若首次出现在表头、阶段列、mermaid 节点或标题中，也必须就地解释或在图表前说明，不能只依赖正文后文补充。
- 抽象层次：ISA、ABI、计算机组织/微结构等名词需说明位置、可见性、相邻层边界，必要时配位置图或表。
- 公式变量：性能公式、AMAT、Amdahl 等要写变量来源和影响因素，不只列公式。
- 追问可执行：每条追问必须含 `> **答**：`，且答案能直接指导手算/判题/实验定位。
- 监修自检：写明二轮还要复查什么，而不是笼统写「待 review」。
- 重要章节粒度：核心机制不能只放在一个小节的一两段里，应拆成子知识点，按动机/术语 → 机制 → 图表 → 图后解释 → 解题模板/易错点展开。
- 图示规则：mermaid 前必须有叙述铺垫，mermaid 后必须解释如何读图；图不能承担首次定义关键术语的任务。
- 示例题模板：所有示例题必须包含题面、已知、求、步骤、结果解释和易错提醒；计算题另需列公式和变量来源。
- 引用块规则：偏解释、类比、读图提示、易错提醒、概念边界说明，可用 `> **直观理解：** ...`、`> **读图提示：** ...`、`> **易错提醒：** ...`、`> **边界说明：** ...` 承载，避免正文主线被提示性内容打断。

二轮优先级固定为：**07b、08、05b、06、05、02、04**；其余章节轻量补齐开卷路径与二轮入口。

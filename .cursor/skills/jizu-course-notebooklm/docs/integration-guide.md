## 2. 认知编排原则

### 2.1 先「学什么」，再「怎么学」

每个大模块（如数据表示、指令系统、流水线…）正文开始前，必须有**全景节**：

| 要素 | 要求 |
|------|------|
| 模块解决什么问题 | 一句话 + 与上周衔接 |
| 学完能做什么 | 3–5 条可检验能力（手算浮点数 / 画数据通路 / 分析流水线冲突…） |
| 内部结构预告 | 问题链或 mermaid，让读者看见路线 |
| 自检问题 | 「读完本节你应该能回答…」 |

### 2.2 认知阶梯（由浅入深）

```
L0 定位与动机 → L1 全景地图 → L2 符号与概念 → L3 推导与机制
  → L4 数值验证 → L5 工程流程 → L6 串联
```

整合顺序按**读者认知阶梯**，不是 manifest 采集顺序。

### 2.3 三层叙事（每章必查）

**（1）章级叙事线**——大节开头：一句话说明本节在整体中的作用

**（2）节级「要回答」**——每子节首行明确要解答的问题

**（3）节间衔接**——每节末尾小结并预告下一节

## 3. 计组特定要求

### 3.1 可视化
- 数据通路图、流水线时空图、存储层次图 **必须用 mermaid**
- 鼓励 ASCII art 辅助简单示意图

### 3.2 数值例
- 浮点数 IEEE 754 表示（32/64 位）
- Cache 地址映射（直接映射/组相联计算）
- 流水线 CPI 计算
- 阿姆达尔定律加速比

### 3.3 对比表（计组高频对比）
- RISC vs CISC
- 单周期 vs 多周期 vs 流水线
- 直接映射 vs 组相联 vs 全相联
- 静态预测 vs 动态预测
- 写直达 vs 写回

# 定稿检查清单

> 每份学习指南定稿前，Agent 必须逐项检查。完整规范见 `integration-guide.md`。

## 元信息与课本对照

- [ ] 元信息头含 **主要来源**、**对应课件**、**教材章节**（唐朔飞 + P&H RISC-V）
- [ ] §1.4 **课本与课件速查**表（指南节 ↔ Week ↔ 课件 ↔ 两本教材）
- [ ] §2 各核心节前有 **四列对照表**（课件 / 唐朔飞 / P&H / 课程记录），章§具体
- [ ] 课件名与 `guides/计组课程-16周内容梳理.md` §1.2 一致
- [ ] 文末 **§资料索引** 汇总课件、教材、记录、Lab/Wiki、raw、知识图谱

## 内容完整性

- [ ] 覆盖该模块全部周次的课程内容
- [ ] 与 `guides/计组课程-16周内容梳理.md` 进度同步
- [ ] 所有 batch 的 raw 答案已通读并整合
- [ ] 标注了 NotebookLM 与课纲的偏差（如有）
- [ ] 有实验的周次：§3 Lab 节引用 **26-Arch Wiki** 与 `4_Lab/` 讲义

## 叙事质量

- [ ] §1.1 全景节 + **学完你能**（3–5 条可检验能力）
- [ ] 章级叙事线 + 节级「本节要回答」
- [ ] 重要机制章节已拆成 `####` 子知识点，而不是每个 Week 只用一个粗小节压缩讲完
- [ ] 核心术语、指令、机制首次出现时，先解释用途 / 动机 / 要解决的问题，再进入图表或公式
- [ ] 每 §2.x 末尾有 **小结 → 下一节承接**（末节可接 Lab/下周）
- [ ] ≥3 张 mermaid 图（数据通路/流水线/存储层次/状态机等）
- [ ] Mermaid 图前有文字铺垫，图后说明怎么读图；图不能替代关键概念讲解
- [ ] ≥2 组易混对比表
- [ ] ≥3 处追问块，且每条含 **`> **答**：`**（非仅提示）
- [ ] 偏解释、类比、读图提示、易错提醒、概念边界说明已优先用引用块承载（如 `> **直观理解：** ...`、`> **读图提示：** ...`、`> **易错提醒：** ...`、`> **边界说明：** ...`），未打断定义/机制/步骤主线

## 难点与数值

- [ ] 难点有补充：真值表、状态时序、完整手算步骤（IEEE 754、Cache 拆位、MESI 迁移等）
- [ ] 示例题含题面、已知、求、步骤、结果解释和易错提醒；计算题另含公式和变量来源
- [ ] 公式正确，符号一致
- [ ] RISC-V 汇编 / CSR 位域与 Lab 一致

## 技术准确

- [ ] IEEE 754、页表遍历、AMAT 等数值例验证无误
- [ ] 数据通路图与控制信号对应正确
- [ ] 标明了参考来源（课件编号/教材章节/batch-id）

## 格式

- [ ] 术语表或“缩写速查”已解释高频缩写；正文首次出现处采用 **中文（English）** 或 **缩写（English full name，中文）**
- [ ] 表格字段、标题、mermaid/图节点中的缩写也已在表头、注释、图前说明或同一表格内给出全称 / 中文解释；没有只在正文解释而忽略图表
- [ ] Markdown 排版正确
- [ ] Mermaid 语法可渲染
- [ ] 代码块标注语言（asm / verilog / python / bash）
- [ ] 没有未替换的占位符（TODO / xxx）

## 用户确认

- [ ] 用户已 Review 并反馈
- [ ] 反馈已整合

## 课件指南定稿

> 适用于 `guides/计组-课件{NN}-学习指南.md`。完整规范见 `integration-guide.md` §8。

### 结构与覆盖

- [ ] 指南元信息头含 `guides/计组-课件学习指南监修标准.md` 链接与首轮监修状态
- [ ] 元信息头含课件 PDF、NotebookLM source、课堂 Week、唐朔飞 + P&H 章§、周次指南链接
- [ ] **课件内容覆盖索引**无遗漏（discovery 每个板块一行，均已映射到 Part·块）
- [ ] Part 正文严格按课件原序；块内含课件要点 → 讲解 → 难点补充
- [ ] 「与周次指南对照」表已填写
- [ ] `guides/计组-课件梳理索引.md` §五进度已更新

### 来源与交叉引用

- [ ] discovery + deep 的 raw 已通读并整合
- [ ] 每块标注 batch-id 或课件来源
- [ ] Lab4–6 相关 Part 已交叉 Wiki / `26-Arch/Doc/Lab*/report.md`
- [ ] 与周次指南无矛盾；冲突处以课纲 + raw 为准并标注

### 质量（与周次轨同标准）

- [ ] 文末含「监修自检（首轮）」或二轮 review 建议
- [ ] ≥2 组易混对比表（如单周期 vs 多周期、转发 vs 阻塞）
- [ ] 数据通路/FSM 等须 mermaid 或等价图示
- [ ] 难点有完整手算或真值表（控制信号、CPI 等）
- [ ] 复习优先级表已按期末权重标注

### 迭代升级（二轮 Review）

- [ ] 已检查是否需要新增「本章怎么用 / 开卷复习路径」，尤其是 07b、08、05b、06、05、02、04。
- [ ] 已说明 **课件顺序 vs 授课顺序** 的差异；必要时指向周次指南。
- [ ] 已补齐 **课件 PDF - 双教材 - Lab/周次指南** 三方定位，不只写「相关」。
- [ ] 难点处有直观解释、反例、边界条件、数值例、状态机或 mermaid 之一；存储/异常/TLB/一致性/ILP 重点不得只列术语。
- [ ] 计算示例题不是只给答案；已写清题目场景、已知、求、公式、逐步计算、结果解释和易错提醒。
- [ ] 重要章节已拆成子知识点，图前有叙述，图后有读图说明；关键术语/指令首次出现已解释用途和被拿来讲的原因。
- [ ] 读图提示、直观类比、易错提醒、边界说明等提示性内容已用引用块标签收束，正文仍按课件原序讲定义、机制和步骤。
- [ ] 缩写/专名首次出现有英文全称、中文解释和课程用途；辅助案例说明与主线 ISA/Lab 的关系。
- [ ] 表格、图节点、标题中的缩写也视为首次出现，已在图表附近补全解释入口。
- [ ] 抽象分层名词（ISA、ABI、计算机组织/微结构等）已说明位置、可见性和相邻层边界。
- [ ] 性能公式等定量公式已说明变量来源、影响因素和常见误读点。
- [ ] 追问答案可操作：每条追问含 `> **答**：`，且答案能指导手算、判题或实验定位。
- [ ] 「监修自检」含二轮待复查项；P0 章节（07b、08、05b、06）明确建议用户重点 review 的机制。
- [ ] 本轮发现的可复用规则已回写到 `integration-guide.md` §9 或 `guides/计组-课件学习指南监修标准.md`，而不是只停留在单章修改。
- [ ] `guides/计组-课件梳理索引.md` 的监修状态 / 二轮 Review 入口已同步。

### 格式

- [ ] 课件指南含统一 **术语格式** 元信息；高频缩写在“缩写速查”或首次出现处有解释入口
- [ ] 术语表或“缩写速查”已解释高频缩写；正文首次出现处采用 **中文（English）** 或 **缩写（English full name，中文）**

- [ ] 无未替换占位符（「待 discovery」等应已替换）
- [ ] Mermaid 可渲染；代码块标注语言

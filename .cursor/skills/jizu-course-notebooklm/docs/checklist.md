# 定稿检查清单

> 每份学习指南定稿前，Agent 必须逐项检查。完整规范见 `integration-guide.md`。

## 元信息与课本对照

- [ ] 元信息头含 **主要来源**、**对应课件**、**教材章节**（唐朔飞 + P&H RISC-V）
- [ ] §1.4 **课本与课件速查**表（指南节 ↔ Week ↔ 课件 ↔ 两本教材）
- [ ] §2 各核心节前有 **四列对照表**（课件 / 唐朔飞 / P&H / 课程记录），章§具体
- [ ] 课件名与 `guides/计组课程-16周内容梳理.md` §1.2 一致
- [ ] 文末 **§资料索引** 汇总课件、教材、记录、Lab/Wiki、raw、知识图谱

## 内容完整性

- [ ] 覆盖该模块全部周次的课程内容
- [ ] 与 `guides/计组课程-16周内容梳理.md` 进度同步
- [ ] 所有 batch 的 raw 答案已通读并整合
- [ ] 标注了 NotebookLM 与课纲的偏差（如有）
- [ ] 有实验的周次：§3 Lab 节引用 **26-Arch Wiki** 与 `4_Lab/` 讲义

## 叙事质量

- [ ] §1.1 全景节 + **学完你能**（3–5 条可检验能力）
- [ ] 章级叙事线 + 节级「本节要回答」
- [ ] 每 §2.x 末尾有 **小结 → 下一节承接**（末节可接 Lab/下周）
- [ ] ≥3 张 mermaid 图（数据通路/流水线/存储层次/状态机等）
- [ ] ≥2 组易混对比表
- [ ] ≥3 处追问块，且每条含 **`> **答**：`**（非仅提示）

## 难点与数值

- [ ] 难点有补充：真值表、状态时序、完整手算步骤（IEEE 754、Cache 拆位、MESI 迁移等）
- [ ] 公式正确，符号一致
- [ ] RISC-V 汇编 / CSR 位域与 Lab 一致

## 技术准确

- [ ] IEEE 754、页表遍历、AMAT 等数值例验证无误
- [ ] 数据通路图与控制信号对应正确
- [ ] 标明了参考来源（课件编号/教材章节/batch-id）

## 格式

- [ ] Markdown 排版正确
- [ ] Mermaid 语法可渲染
- [ ] 代码块标注语言（asm / verilog / python / bash）
- [ ] 没有未替换的占位符（TODO / xxx）

## 用户确认

- [ ] 用户已 Review 并反馈
- [ ] 反馈已整合
